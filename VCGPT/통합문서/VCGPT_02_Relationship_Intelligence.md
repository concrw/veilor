## 1. 관계 인텔리전스의 정의

Relationship Intelligence는 다음 5가지 능력을 포함한다:

1. 자신의 감정적·심리적 패턴을 이해한다
2. 타인의 행동과 니즈를 해석한다
3. 욕망과 경계를 명확히 소통한다
4. 서로에게 유익한 관계 구조를 만든다
5. 갈등과 변화를 시간 속에서 항해한다

대부분의 사람은 이 능력에 대해 구조적 교육을 받지 못한다.
Veilrum은 **관계 인텔리전스를 위한 지속적 학습 시스템**으로 기능한다.

## 2. Veilrum이 정의하는 관계 실패 구조

### 관계 문제를 보는 두 가지 방식

**일반적 관계 담론:**
- 저 사람이 문제다
- 내가 잘못 선택했다
- 우리는 안 맞는다
- 사랑이 식었다
- 소통이 안 된다

**Veilrum의 구조적 관점:**
- 어떤 페르소나가 누구 앞에서 올라오는가
- 내 욕망과 상대 욕망은 어디서 어긋나는가
- 나는 왜 반복적으로 비슷한 유형에 끌리는가
- 어떤 감정이 실제 욕망을 가리고 있는가
- 무엇이 말해졌고, 무엇이 끝내 말해지지 않았는가

Veilrum은 관계를 사건이 아니라 **구조와 패턴**으로 읽는다.

### 관계 실패의 근본 원인

> **Most relationships fail because people never see the veil.**

사람들은 보통:
- 가면과 가면이 만난다
- 욕망을 숨긴 채 관계를 시작한다
- 실제 구조를 말하지 않는다
- 기대를 명확히 하지 않는다
- 시간이 지나고 나서야 진짜 구조를 보게 된다

따라서 많은 관계는 악의 때문이 아니라 **보지 못한 것** 때문에 실패한다.

### 관계력 방정식 (밀라르쉬)

**관계력 = 관찰력 + 이해력 + 소통력**

**변화 = 깨달음 × 각오 × 반복실행** (곱셈 — 하나라도 0이면 결과 0)

## 3. 관계 라이프사이클 모델 6단계

전통 플랫폼은 연결 단계까지만 다룬다.
Veilrum은 전체 사이클을 다룬다.

| 단계 | 이름 | 내용 |
|------|------|------|
| 1 | Self Understanding | 나는 누구인가, 내 패턴은 무엇인가 |
| 2 | Understanding Others | 타인의 행동과 구조를 해석한다 |
| 3 | Alignment & Matching | 욕망·기대·가치관이 정렬된 사람과 연결 |

## 4. 자기이해 → 연결 → 경험 → 성장 순환 루프

Veilrum의 가장 간단한 구조 요약:

### 4.1 자기이해
사용자는 먼저 자신을 읽는다.
- 내가 어떤 사람인지
- 내가 어떤 패턴을 보이는지
- 내가 무엇을 두려워하고 원하는지
- 내 안의 어떤 페르소나가 지금 전면에 나와 있는지

### 4.2 연결
그다음 같은 방향, 같은 구조, 같은 욕망을 가진 사람과 연결된다.
중요한 것은 단순 취향 일치가 아니다:
- 관계 구조 정렬
- 욕망 정렬
- 정서적 안전감 정렬
- 자극의 수준 정렬
- 친밀감 방식 정렬

→ **외모/거리/스펙 기반 연결이 아니라 관계 정렬 기반 연결**

### 4.3 경험
연결된 사람은 실제로 관계를 경험한다.
- 원하는 대화 / 안정감 / 자극 / 친밀감 / 섹스 / 관계 구조

### 4.4 성장
경험은 다시 자기이해를 심화시킨다.
- 나는 여기서 왜 불안했는가
- 왜 이 사람 앞에서 더 솔직해졌는가
- 나는 어떤 관계 구조에서 가장 평화롭고 생생한가

→ **일회성 관계 서비스가 아니라 반복적으로 더 정확해지는 관계 학습 플랫폼**

## 5. Veil 3층 구조 — 모든 관계에 있는 베일

```text
표면: Persona Layer — 가면 / 페르소나
↓
중간: Desire Layer — 숨겨진 욕망
↓
깊이: Relationship Structure Layer — 실제 관계 구조
```

### 5.1 Persona Layer (가면)
사람은 관계마다 다른 얼굴을 쓴다.
- 직장인의 얼굴 / 연애할 때의 얼굴
- 좋은 사람인 척하는 얼굴 / 욕망 없는 척하는 얼굴
- 상처받지 않은 척하는 얼굴

가면이 너무 굳어지면 자기 자신도 잘 못 보게 된다.

### 5.2 Desire Layer (숨겨진 욕망)
가면 아래에는 욕망이 있다.
- 사랑받고 싶은 욕망 / 인정받고 싶은 욕망
- 지배하고 싶은 욕망 / 보호받고 싶은 욕망
- 강한 자극을 원하는 욕망 / 성적 욕망

대부분의 사람은 이 욕망을 숨기거나, 미화하거나, 스스로도 명확히 보지 못한다.

### 5.3 Relationship Structure Layer (실제 관계 구조)
가장 깊은 층에는 관계의 실제 구조가 있다.
- 사랑처럼 보이지만 실은 의존 구조
- 돌봄처럼 보이지만 실은 권력 구조
- 헌신처럼 보이지만 실은 자기희생 구조
- 보호처럼 보이지만 실은 통제 구조

### 5.4 세 층이 왜 하나처럼 느껴지는가
세 층은 실제 삶에서 늘 함께 움직인다.
사람들은 보통 표면(Persona)만 보고 관계를 시작한다.
시간이 지나면 욕망(Desire)이 드러나고, 결국 실제 구조가 모습을 드러낸다.

> **Veilrum은 이 베일을 보게 만드는 플랫폼이다.**

## 6. 관계 패턴 분석 — 핵심 진입점

### 6.1 가장 강력한 한 질문

모든 관계 문제(연애, 이별, 자기이해)는 결국 하나의 상위 질문으로 수렴한다:

> **"나는 왜 이런 관계를 반복하는가?"**

이 질문이 Veilrum의 가장 강력한 진입점이다.

### 6.2 연애 문제도 결국 패턴이다
연애 문제는 보통 사건처럼 느껴진다:
- 연락 문제 / 집착 / 거리두기 / 감정 기복 / 질투

하지만 Veilrum은 이것을 **관계 패턴이 드러난 장면**으로 본다.

### 6.3 7가지 Relationship Pattern System (상세)

Veilrum이 실제 서비스에서 가장 많이 다뤄야 하는 것은 테스트 결과 자체보다 **반복 패턴**이다.

**"당신은 이런 사람"이 아니라 "당신은 이런 장면에서 이런 패턴을 반복한다"를 말해야 한다.**

#### Attraction Pattern — 끌림 패턴
- 왜 이런 사람에게 끌리는가
- 익숙한 불안에 끌리는가
- 안전한 사람을 지루하게 느끼는가
- 구원 대상에게 끌리는가

#### Attachment Pattern — 애착 패턴
- 가까워질수록 불안해지는가
- 멀어질수록 집착하는가
- 깊어질수록 회피하는가
- 사랑받고 싶지만 삼켜질까 두려운가

#### Conflict Pattern — 갈등 패턴
- 직면형 / 참다 폭발형 / 침묵 처벌형 / 논리 지배형 / 추격-회피형

#### Power Pattern — 권력 패턴
- 명시적 주도 / 숨은 지배 / 돌봄 권력 / 철수 권력 / 성적 권력 / 정보 권력

#### Sexual Pattern — 성적 패턴
- 친밀감 중심형 / 자극 중심형 / 확인 추구형 / 통제형 / 회피형 / 수치-욕망 충돌형

#### Care Pattern — 돌봄 패턴
- 과잉 돌봄형 / 구원형 / 돌봄 회피형 / 돌봄 의존형 / 돌봄 거래형

#### Boundary Pattern — 경계 패턴
- 느슨한 경계 / 과잉 경계 / 지연 경계 / 상황 붕괴형

### 6.4 기능적 자아 유형 8가지

멀티페르소나 모델에서 실제 관계 속에서 작동하는 기능적 자아:

| 기능적 자아 | 핵심 |
|-----------|------|
| 보호자 자아 | 상대를 지키려는 자아 — 과잉 시 돌봄 권력으로 변질 |
| 통제자 자아 | 상황을 장악하려는 자아 |
| 도망자 자아 | 위험에서 철수하려는 자아 |
| 구원자 자아 | 상대를 고치려는 자아 — SAV 가면과 연결 |
| 인정갈구 자아 | 인정을 얻으려는 자아 — APV 가면과 연결 |
| 매혹자 자아 | 상대를 사로잡으려는 자아 |
| 실험자 자아 | 새로운 것을 시도하려는 자아 |
| 관찰자 자아 | 거리를 두고 보려는 자아 |

AI는 다음을 읽어야 한다:
- 지금 말하고 있는 건 어떤 자아인가
- 어떤 트리거가 그 자아를 켰는가
- 그 자아는 무엇을 보호하거나 얻고 싶은가

### 6.5 딥다이브 3단계 (밀라르쉬 방법론)

**정의 공유 → 각인 추적 → 각인 원인 분석**

1. **정의 공유:** "나에게 [단어]란?" — 각자가 같은 단어를 어떻게 다르게 정의하는지
2. **각인 추적:** "왜 그렇게 정의하게 됐나?" — 경험의 기원
3. **원인 분석:** "그 경험은 어디서 왔나?" — 패턴의 뿌리

## 7. 관계 플랫폼이 반드시 해결해야 하는 구조적 문제 9가지

### 문제 1 — "연결"이 곧 "좋은 관계"가 아니다

연결 중심 플랫폼은 시간이 지나면 이렇게 된다:
- 반복적인 얕은 만남 / 같은 문제의 재생산 / "사람은 많은데 관계는 없다"

**Veilrum의 방향:** 연결을 목표가 아니라 **정렬된 관계 형성의 중간 단계**로 봐야 한다.

성공 지표: 매칭 수 → 서로의 관계 기대를 얼마나 일찍 명확히 했는가, 관계 질이 개선되었는가

### 문제 2 — 사람들은 자기 자신을 잘 모른다

사람들은 종종:
- 자신이 원하는 사람을 안다고 생각하지만 패턴만 반복한다
- 자신이 솔직하다고 생각하지만 핵심 욕망은 숨긴다
- 안전을 원한다고 말하지만 실제론 강한 자극에 끌린다

**Veilrum의 방향:** 매칭 이전에 **사용자가 자기 자신을 더 잘 알게 만드는 엔진**이 필요하다.

### 문제 3 — 욕망과 관계 구조는 초기에 잘 말해지지 않는다

사람들이 핵심을 숨기는 이유:
- 거절당할까 봐 / 이상해 보일까 봐 / 자기 욕망을 언어화할 수 없어서

결과: 기대 불일치, 관계 구조 충돌, 늦게 드러난 진실로 인한 파탄

**Veilrum의 방향:** 합의와 정렬을 촉진하는 대화 인터페이스, 단계별 개방 구조

### 문제 4 — 정보 비대칭이 너무 크다

누구나 자신을 꾸밀 수 있다. 플랫폼에 이를 걸러낼 구조가 없으면:
- 신뢰 하락 / 냉소화 / 외모/스펙 중심 회귀

**Veilrum의 방향:** 자기소개 문장보다 **행동 기반, 시간 기반 데이터** — 반복 기록 일관성, 대화 성향, 변화 리포트

### 문제 5 — 커뮤니티는 쉽게 소음과 극단성으로 무너진다

관계/성/욕망/상처는 감정 에너지가 크기 때문에 익명 커뮤니티는 빠르게 극단화된다.
- 자기확증 / 낙인찍기 / 성급한 진단 / 집단 공격

**Veilrum에 특히 위험한 이유:** 포식/피식, 빌런/히어로, 비밀스러운 나 같은 주제가 품질 관리 실패 시 불안 증폭 공간으로 변질

**Veilrum의 방향:** 기능보다 운영 설계 먼저, "진단"보다 "경험 공유"를 유도하는 구조

### 문제 6 — 안전과 자유는 항상 긴장 관계다

자유만 강조 → 위험 인물 유입, 조작/착취
안전만 강조 → 생동감 저하, 진짜 욕망 표현 위축

**Veilrum의 방향:** "자유 vs 안전"이 아니라 **합의 기반 자유** 설계
- 경계 설정 도구 / 합의 전 체크리스트 / 신뢰 레벨별 기능 개방 / 철회 가능성 보장

### 문제 7 — 관계 유지와 성장은 연결보다 훨씬 어렵다

관계가 시작되고 나면 등장하는 것들:
- 오해, 권태, 질투, 거리감, 욕망 변화, 시간에 따른 페르소나 변화

플랫폼이 못 풀면: 계속 새 사람만 추천 → 관계를 성장시키기보다 새 연결로 문제를 덮는 방식에 익숙해짐

**Veilrum의 방향:** 관계 체크인, 갈등 해석 AI, 합의 재설정 도구, 관계 단계별 코칭 → **관계 유지 플랫폼**

### 문제 8 — 네트워크 효과가 반드시 좋은 방향으로만 작동하지 않는다

좋은 증가: 더 다양한 관계 가능성, 더 풍부한 데이터
나쁜 증가: 더 많은 저품질 유입, 더 빠른 커뮤니티 악화

**Veilrum의 방향:** "많이 모이는 것" 자체가 목표가 아니라 **어떤 문화가 강화되는가** → 양적 네트워크보다 **문화적 네트워크 효과** 설계

### 문제 9 — 유형화는 유용하지만, 낙인으로 변질되기 쉽다

유형 언어는 빠르게 설명되고 대화를 시작하게 하지만, 사람을 고정시키고 상대를 낙인찍기 쉽다.

**Veilrum에 특히 중요한 이유:** 포식/피식, 빌런/히어로, 가면, 변화준비도 같은 강한 언어를 가짐

**Veilrum의 방향:**
- 정체성 규정이 아니라 **현재 패턴 가설**
- 영구 성질이 아니라 **맥락 기반 경향**
- "당신은 DEP입니다" (X) → "DEP 경향이 반복적으로 관찰됩니다" (O)

## 8. Sex / 금기 영역 — RI의 핵심 도메인

Veilrum에서 sex는 부가 기능이 아니라 **Relationship Intelligence의 핵심 도메인**이다.

### 왜 Sex 영역이 핵심인가

sex 영역은 다음의 농축 지점이기 때문이다:
- 친밀감 / 수치심 / 욕망 / 권력 / 애착 / 합의 / 경계 / 노출

관계의 깊은 문제는 종종 이 영역에서 터진다. sex를 빼면 관계를 반만 이해하게 된다.

### 사람들이 실제로 겪는 문제

| 영역 | 실제 문제 |
|------|---------|
| 욕망 언어화 실패 | 뭘 원하는지 모르겠다 / 알지만 말할 수 없다 / 말하면 이상해보일까 무섭다 |
| 성적 불일치 | 욕망 빈도 차이 / 구조 선호 차이 / 역할 선호 차이 / 친밀감-sex 연결 방식 차이 |
| 수치심과 억압 | 하고 싶은 게 있는데 부정한다 / 하고 싶지 않은데 맞춰준다 |
| 권력과 경계 | 거절이 안전하지 않다 / 동의와 압박이 섞여 있다 |
| 구조적 오해 | "사귀면 당연히" / "결혼했으면 당연히" 같은 문장이 실제 합의를 가린다 |

### Veilrum의 원칙

1. **판단보다 해석** — 성적 욕망을 즉시 도덕화하지 않고, 무엇이 반복되고 무엇이 충돌하는지를 먼저 읽는다
2. **금기 회피 금지** — 비독점/배타성, BDSM, 돈과 sex, 성적 권력, 역할 갈등, 수치심/판타지, sexless, 불일치와 외도를 구조적으로 다룬다
3. **경계는 분명히** — 합의·안전·합법성·철회 가능성 기준 밖 내용 불허
4. **sex는 관계 맥락 안에서** — 기술 조언이 아니라 이 사람의 애착/욕망/수치심/권력 구조 안에서 읽는다

### Sexual Relationship Intelligence 6차원

| 차원 | 내용 |
|------|------|
| Orientation | 누구에게 끌리는가 (무성애/반성애/이성애/양성애/범성애) |
| Structure | 어떤 관계 구조를 원하는가 (모노가미/ENM/폴리아모리/캐주얼) |
| Role | 권력 구조 (DOM/SUB/SWITCH/Vanilla/control-seeking/reassurance-seeking) |
| Desire Frequency | 욕망 빈도 (low/medium/high/mismatch-sensitive) |
| Intimacy Link | 친밀감-sex 연결 방식 (감정 선행형/분리형/자극 중심/확인 중심/수치-욕망 충돌형) |
| Sexual Communication | 말할 수 있는가 (경계/거절/욕망 공유/애프터케어/트리거 공개 가능성) |

## 8-1. AI 해석 5원칙

### 원칙 1 — 고정 진단 금지
"당신은 회피형이다" (X)
→ "깊어질수록 거리를 두는 패턴이 반복적으로 보인다" (O)

### 원칙 2 — 단일 라벨 금지
한 사람은 여러 패턴을 동시에 가진다.

### 원칙 3 — 맥락 포함
누구 앞에서, 어떤 상황에서, 어떤 욕망이 걸려 있을 때 나타나는지 함께 봐야 한다.

### 원칙 4 — 변화 가능성 포함
패턴은 운명이 아니라 현재 반복되는 구조다.

### 원칙 5 — 금기 영역에서도 동일한 원칙
성적 욕망이나 수치심 역시 낙인이 아니라 해석과 정렬의 대상이어야 한다.

## 9. Veilrum 플랫폼 설계 원칙 6가지

위 구조 문제들을 정리하면 Veilrum은 다음 원칙을 가져야 한다:

### 8.1 연결보다 정렬
사람을 많이 연결시키는 것보다 맞는 구조의 관계를 만들게 하는 것이 중요하다.

### 8.2 프로필보다 축적 데이터
자기소개보다 시간축 데이터가 더 중요하다.

### 8.3 자유보다 합의 기반 자유
무제한 자유도, 과잉 통제도 아닌 합의 구조가 핵심이다.

### 8.4 진단보다 대화 촉진
유형을 알려주는 것보다 그 유형 언어를 통해 더 좋은 대화를 하게 하는 것이 중요하다.

### 8.5 매칭보다 관계 유지
새로운 연결보다 관계를 더 잘 유지하고 발전시키게 하는 것이 차별점이다.

### 8.6 대규모보다 문화 품질
사람 수보다 문화의 질과 신뢰 구조가 더 중요하다.

## 9. 관계 위험 신호 분류

AI 엔진이 감지해야 할 핵심 위험 신호:

### 즉각 위험 신호 (Urgent)
- 신체적 위협 / 강압적 통제 / 자해 위험 언급

### 구조 위험 신호 (Pattern Risk)
- 반복적 트라우마 본딩 패턴
- 성적 경계 반복 침범
- 권력 비대칭 + 의존 구조 결합
- 공동의존성 강화 패턴

### 합의 위험 신호 (Consent Risk)
- 욕망 표현 불일치 + 회피
- 관계 구조 기대 충돌 (독점 vs 개방)
- 성적 속도/스타일 불일치

### 발전 위험 신호 (Growth Block)
- 동일 패턴의 3회 이상 반복
- 변화준비도 0 상태 지속
- 자기이해 정체 (기록 중단, 같은 질문 반복)

---

> **최종 한 문장:**
> 사람을 만나게 하는 것만으로는 충분하지 않다. 사람들 사이에 더 좋은 관계가 실제로 생기게 해야 한다.

## 9. Sexual Intelligence Framework (성 호환성 5차원)

### 9.1 성적 호환성 5가지 핵심 차원

| 차원 | 내용 |
|------|------|
| **Desire** | 욕망의 종류, 강도, 빈도 |
| **Communication** | 성에 대해 말할 수 있는가 |
| **Boundaries** | 각자의 경계를 인식하고 존중하는가 |
| **Emotional Meaning** | 성행위에 부여하는 감정적 의미가 같은가 |
| **Compatibility** | 전체적 성적 궁합 |

### 9.2 성적 불일치 유형 (4가지)

- **frequency_mismatch** — 원하는 빈도의 차이
- **style_mismatch** — 선호하는 방식/스타일의 차이
- **initiative_mismatch** — 누가 주도하느냐의 차이
- **emotional-sex mismatch** — 성행위에 부여하는 감정 의미의 차이 (한 사람은 친밀감, 한 사람은 쾌락)

### 9.3 AI 성상담 4원칙

1. **Non-judgmental** — 판단 없이
2. **Consent aware** — 합의 인식
3. **Encourage communication** — 커뮤니케이션 장려
4. **Reduce shame** — 수치심 감소

## 10. Sexual Dialogue System Design

### 10.1 문제 인식

성에 대한 깊은 대화를 할 수 있는 공간이 거의 없다. 사람들이 이야기하지 못하는 것:
- sexual mismatch
- fantasy
- desire shame
- consent confusion

Veilrum은 이 공간을 만든다.

### 10.2 대화 4레이어 (진입 구조)

| 레이어 | 목적 |
|-------|------|
| Layer 1 — Emotional Safety | 판단받지 않는다는 심리적 안전 확보 |
| Layer 2 — Curiosity Exploration | 자신의 욕망과 호불호 탐색 |
| Layer 3 — Desire Articulation | 원하는 것을 언어로 표현 |
| Layer 4 — Relationship Context | 현재 관계 맥락 안에서의 해석 |

### 10.3 핵심 탐색 질문 3개

- "말하고 싶은데 말하지 못하는 게 있나요?"
- "친밀감에서 지금 무엇이 빠져 있는 것 같나요?"
- "물어보기 불안한 부분이 어디인가요?"

### 10.4 Safety Rules

AI는 다음을 하지 않는다: judgement / shaming / moralizing

### 10.5 추출 신호

desire_frequency_gap / sexual_shame / cannot_discuss_desire / consent_ambiguity

### 10.6 관계 통합

Sexual data는 relationship signals(attachment/boundary/power/desire)과 함께 분석된다 — 성 문제를 독립적 이슈가 아닌 관계 구조의 일부로 본다.
