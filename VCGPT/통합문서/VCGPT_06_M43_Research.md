## 1. M43 설립 Why & 선언

### 설립 목적

**M43 Institute** — 밀라르쉬(Mille Arche)가 설립한 관계 연구 기관.

**"관계의 모든 것"을 실증 데이터로 연구하고, 밀라르쉬의 이론 체계를 검증·확장한다.**

인간은 세 개의 층으로 관계를 맺는다:

| 층 | 관계 형태 | 결과 |
|----|---------|------|
| 사회적인 나 끼리 | 연애 | 설렘 |
| 일반적인 나 끼리 | 결혼 | 현실, 지지고 볶음 |
| 비밀스러운 나 | 평생 숨김 | 고독 |

두 사람이 함께 사는데 각자의 1/3은 영원히 혼자다. 대부분은 "원래 그런 거"라고 받아들인다.

밀라르쉬는 여기에 동의하지 않는다.

> "헤어질 용기가 없으면 속사랑은 불가능하다."
> "더 깊은 곳의 나와 너가 사랑하려면, 헤어질 용기와 미움 받을 용기가 필요하다."

### 핵심 전제

- 합의(Consensus)의 시작은 **내 안의 멀티페르소나들 간의 합의**다.
- 합의가 된 사람은 후회가 없다. 합의 없이 달린 사람은 후회가 많다.
- 231개 연구 도메인은 이 전제의 실증 데이터를 생산하기 위해 설계되었다.
- **산출물의 최종 목적지:** Veilrum 플랫폼의 알고리즘과 콘텐츠 기반

### Relationship Intelligence 선언

인간 관계는 행복·정신 건강·경제적 안정·커뮤니티 회복력을 결정하지만, 관계 지능(Relationship Intelligence)은 아직 체계적으로 구조화되지 않았다.

Veilrum과 M43은 **관계의 체계적 과학**을 제안한다. 통합 학문:
- 심리학 / 행동과학 / 사회학 / 네트워크 과학 / 인공지능

목표: 개인이 자신을 이해하고, 호환되는 파트너를 선택하고, 갈등을 탐색하며, 지속 가능한 커뮤니티를 구축할 수 있는 도구.

> Veilrum은 단순한 앱이 아니다. 인간 관계의 체계적 과학을 구축하려는 시도다.

## 2. 멀티페르소나 합의 모델

**밀라르쉬의 핵심 이론. M43 연구 전체의 이론적 기반.**

### 전제

인간 안에는 여러 개의 페르소나가 존재한다. 이들은 자리를 놓고 경쟁하며, 하나가 오랫동안 전면에 나오면 나머지는 억압된다.

**대표 사례:**
- 평생 1번 페르소나로 살다 명퇴 후 → 1의 자리가 비자 3·4·5가 난리 → 속은 대환장파티
- 정상에 올라놓고 갑자기 스스로 세상을 떠남 → 1번 붕괴 = 자기 전체 붕괴로 인식
- 잘 살아놓고도 이유 모를 무기력 → 억압된 페르소나들의 반란

*이것은 은유가 아니다.* fMRI에서 역할 수행·방어 해제 시 뇌 활성화 패턴이 완전히 다르다. 1번 페르소나의 퇴장 = 사별과 동일한 신경 반응. (나디아 코소비치)

### 합의 (Consensus)

**합의 = 각 페르소나가 독립적으로 존재하면서 공존 조건을 협정하는 것**

통합이 아니다. 조화도 아니다. 수평적 협상이다.

| | 합의한 사람 | 합의 없이 달린 사람 |
|--|------------|------------------|
| 1번 페르소나 붕괴 시 | 3·4·5가 안전망 | 자기 전체 붕괴로 인식 |
| 결과 | 후회가 없다 | 후회, 무기력, 극단 선택 |

### 세 층위 × 상황 작동 구조

```
공식: [세 층위] × [상황(사람·장소·분위기)] = 현재 드러나는 페르소나
```

- **사회적인 나**: 처음 만났을 때 보여주는 모습
- **일반적인 나**: 친해지면 보여주는 모습
- **비밀스러운 나**: 깊은 신뢰 관계에서만 공유되는 모습 (욕망, 두려움, 상처)
- 무엇이 열리고 닫히는지를 결정하는 것: 그 사람이 뭘 갈망하고, 뭘 포기했고, 뭐에 실망했느냐

### 선행 이론과의 차별점

| 학자 | 이론 | 한계 | 밀라르쉬와 차이 |
|------|------|------|--------------|
| 융 | 페르소나 vs 그림자 | 역동적 경쟁 구조 없음 | 2개 대립 → n개 경쟁 |
| 고프만 | 전면/후면 자아 | 사회학적 관찰 | 내면 구조·권력 역동 |
| 조하리 윈도우 | 인지 인식 축 | 수평적 협상 구조 없음 | — |
| 슈워츠 (IFS) | 내면 파트 + Self 중재 | 치료 모델, 수직 위계 | 수평적 합의, 인생 설계 |

**3가지 핵심 차별점:**
1. 페르소나들의 **권력 역동** (현실 사건이 트리거)
2. 통합/조화가 아닌 **합의** — 각각 독립적으로 존재하면서 공존 조건 협정
3. **후회의 메커니즘**으로 연결 → 인생 설계의 기반

"비밀스러운 나"를 별도 층위로 분리하고 관계의 최종 깊이를 결정하는 것으로 본 이론은 선행 연구에서 확인 안 됨.

### 밀라르쉬 핵심 철학 공식

- **변화 = 깨달음 × 각오 × 반복실행** (곱셈 구조 — 하나라도 0이면 결과 0)
- **관계력 = 관찰력 + 이해력 + 소통력**
- **딥다이브 3단계:** 정의 공유 → 각인 추적 → 각인 원인 분석

### 논쟁 이력 — 포지티브/네거티브 레드펜

연구팀 그룹헤드 4인이 극단적 포지티브·네거티브 시나리오를 발표하고 상호 공격했다.
- 포지티브의 약점: "아직 안 됐다 (not yet)" — 검증과 실행의 문제
- 네거티브의 약점: "안 된다고만 했다 (only said no)" — 솔루션 부재
- **모든 분기점이 검증과 통제에 달려 있음** → M43의 존재 이유

## 3. 연구 스코프 — 발견·관리·만족

**231개 도메인 × 발견·관리·만족 = 693개 연구 셀. 이것이 M43 연구의 전체 스코프다.**

| 단계 | 실패 현상 | M43의 역할 | Veilrum 기능 |

나디아 계산: 231개 도메인 × 3단계 = **693개 연구 셀**

### 5-3. V/H — 빌런-히어로 프레임워크

같은 가면이라도 빌런으로 굳거나 히어로로 성장한다.

- **빌런:** 가면이 자아로 굳음 → 무의식에 지배
- **히어로:** 가면을 의식적으로 작동 → 무의식을 의식으로 끌어올림

**V/H 이동을 결정하는 3요소:** 자기인식 × 변화준비도(CHG) × 안전한 관계 경험

---

### 5-4. SEX — 성적 스펙트럼 3축 (최종 확정)

- **Leading** (주도성) -1.0 ~ +1.0
- **Expressiveness** (표현성) -1.0 ~ +1.0
- **Intensity** (강렬함) -1.0 ~ +1.0

**측정:** 자가 보고 + 파트너 평가(익명) + AI 분석 3-Way 교차검증
**문화권 보정 적용:** (문화권 평균 - 개인 점수 = 상대점수)
**자가-파트너 간극** = "자기인식 왜곡 지수"로 별도 연구 활용

### 5-5. M/S — 돈-섹스 상관

HAB(아비투스) + SEX(성적 스펙트럼) 완료 후 진행.
경제력과 성적 관계의 연동 패턴 연구. 진화심리학·교환이론 학술 접점.

---

### 5-6. DIV — 이혼 4유형

**4가지 이혼 경로 (가트만 이혼 예측 연구 + 애착이론 기반):**
1. 비난형 — 파트너를 인격적으로 공격

4. 담쌓기형 — 정서적 철수, 무반응

### 5-7. CHG — 변화준비도 5단계

**변화 = 깨달음 × 각오 × 반복실행 (에버렛 로저스 혁신확산이론 적용)**

| 단계 | 비율 | 정의 | 코칭 접근 |

| 이노베이터 | 2~3% | 세상이 두 쪽 나도 본인이 발견한 것을 증명하는 사람 | 방향만 제시 |
| 얼리어답터 | 13~14% | 깨달은 것만큼 빠르게 내 것으로 만드는 사람 | 도구 제공 |

## 8. 연구 생성 3가지 흐름 (Claude 실행 기준)

### 흐름 1 — 연구원 발언/분석

각 연구원이 본인이 lead를 맡은 도메인에 대해 작성.
- **DB 테이블:** `m43_research_outputs`
- **type:** analysis | statement | case_study | hypothesis | field_note | rebuttal
- **예시:** Vera Lindgren(D1 Head)이 "가면 12종 분류"(D1-001)에 대한 분석 리포트 작성

### 흐름 2 — 밀라르쉬 발제 이론

밀라르쉬가 특정 도메인에 대한 핵심 이론·프레임워크를 발제.
- **DB 테이블:** `m43_domain_theories`
- **type:** core_theory | framework | proposition | synthesis
- **author_name:** '밀라르쉬' (m43_researchers에 없음, 의도적 설계)

### 흐름 3 — 밀라르쉬 ↔ Group Head 세션

밀라르쉬가 발제하고 Group Head(또는 Division Head)들과 세션 형태로 토론.
- **DB 테이블:** `m43_sessions` + `m43_session_contents`
- 세션 발언이 정식 이론/분석으로 발전 가능 (derived_theory_id, derived_output_id)

### 계보 연결 구조

```
밀라르쉬 발제 (domain_theories)
  → 연구원 분석 (research_outputs, parent_theory_id)
    → 반론/파생 (research_outputs, response_to_output_id)
      → 세션 토론 (session_contents)
        → 정식 이론으로 승격 (derived_theory_id)
```

### 생성 시 각 연구원 시점 유지

- 발언·분석은 반드시 해당 researcher_id의 전문성·성격에서 나오는 언어로
- Vera Lindgren (스웨덴, 심리학) / Rafael Moreira (포르투갈, 성과학) 등 각 캐릭터 배경 반영
- 밀라르쉬 발제는 선구적·철학적 언어 / 연구원 분석은 학술적·실증적 언어

## 9. INSERT SQL 표준 패턴

### 밀라르쉬 발제 1편 생성 시 표준 순서

```sql
-- STEP 1: 도메인 ID 확인
SELECT id FROM m43_domains WHERE code = 'D1-001';

-- STEP 2: domain_theories INSERT
INSERT INTO m43_domain_theories
  (domain_id, title, content, summary, type, status, tags)
VALUES ('{domain_id}', '제목', '본문(Markdown)', '요약 1~2문장', 'core_theory', 'draft', ARRAY['가면', '페르소나'])
RETURNING id;

-- STEP 3: research_outputs INSERT (연구원 분석, 발제 이론 기반)
INSERT INTO m43_research_outputs
  (researcher_id, domain_id, title, content, summary, type, status, parent_theory_id, tags)
VALUES ('{researcher_id}', '{domain_id}', '제목', '본문', '요약', 'analysis', 'draft', '{theory_id}', ARRAY['가면']);

-- STEP 4: sessions + session_contents (세션 토론)
INSERT INTO m43_sessions (title, type, topic, domain_ids, participant_ids, theory_ids, status)
VALUES ('세션명', 'millarche_gh', '주제', ARRAY['{domain_id}'], ARRAY['{gh_id}'], ARRAY['{theory_id}'], 'completed')
RETURNING id;

INSERT INTO m43_session_contents
  (session_id, speaker_name, sequence, content, type, referenced_domain_id, referenced_theory_id)
VALUES ('{session_id}', '밀라르쉬', 1, '발언 내용', 'statement', '{domain_id}', '{theory_id}');
```

### DB 조회 쿼리

```sql
-- 연구원 전체 조회
SELECT * FROM m43_researchers;

-- 도메인 전체 조회 (231개)
SELECT * FROM m43_domains ORDER BY code;

-- 연구 지침 조회
SELECT * FROM m43_domain_research_guide;

-- 특정 Division 이론 조회
SELECT dt.*, d.code FROM m43_domain_theories dt
JOIN m43_domains d ON dt.domain_id = d.id
WHERE d.code LIKE 'D1-%' ORDER BY d.code;

-- 세션 조회
SELECT * FROM m43_sessions ORDER BY created_at DESC;
```
