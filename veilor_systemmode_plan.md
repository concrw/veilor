# VEILOR 시스템모드 — 프론트엔드 기획서

> 작성일: 2026-04-14  
> 근거: 경쟁사 UX 리서치 (`veilor_systemmode_benchmark.md`) + 현재 코드베이스 분석  
> 범위: 세 번째 UX 모드의 프론트엔드 설계. 구현 지시서가 아닌 기획 문서.

## 1. 리서치에서 추출한 공통 UI 패턴

경쟁 6개 앱(Daylio, Headspace, BetterUp, Reflectly, Calm, Woebot)에서 반복적으로 나타나는 프론트엔드 패턴을 정리한다.

### 1-1. 레이아웃

| 패턴 | 근거 앱 | 구체적 특징 |
|------|---------|------------|
| **단일 컬럼 수직 스크롤** | 전체 공통 | 좌우 패딩 16~20px. 요소가 순서대로 쌓임 |
| **카드 기반 섹션 분리** | Headspace, BetterUp, Reflectly | 각 정보 블록이 독립 카드로 분리. 배경과 명확한 경계 |
| **하단 고정 CTA** | Daylio, Reflectly, Calm | 핵심 액션 버튼이 항상 하단에 고정. 스크롤 중 사라지지 않음 |
| **홈 상단에 상태 요약 1개** | Calm(Weekly Goal), BetterUp(코치 카드), Reflectly(슬라이더) | 스크롤 없이 현재 상태 즉각 인식 |
| **탭 3~4개** | 전체 공통 | 하단 탭 3~5개. 홈이 항상 첫 번째 |

### 1-2. 컬러 시스템

| 특징 | 구체적 패턴 | 근거 |
|------|------------|------|
| **다크 배경** | #121212~#1C1C1E 범위 (순수 검정 피함) | 다크모드 표준: subtle dark gray, not pure black |
| **표면 레이어** | 배경보다 10~15% 밝은 카드 배경 | 다크모드에서 그림자 대신 밝기로 계층 표현 |
| **포인트 컬러 1개** | 앱당 시그니처 1개 (Headspace: 주황, Calm: 파랑, Reflectly: 보라) | 2개 이상 쓰면 혼란 |
| **성공/완료 컬러** | 초록 계열 (Headspace: emerald, Daylio: green) | 감정적 의미 일관성 |
| **텍스트 계층 3단** | 주요: white/stone-100, 보조: stone-400, 힌트: stone-600 | 모든 앱 공통 |

### 1-3. 타이포그래피

| 레벨 | 크기 | 무게 | 역할 |
|------|------|------|------|
| **섹션 레이블** | 10~11px, letter-spacing 0.2em, uppercase | 400 | "이번 주 기록" 같은 카테고리 |
| **카드 제목** | 14~16px | 600 | 카드 주요 내용 |
| **핵심 수치** | 24~32px | 700, tabular-nums | 숫자 지표 |
| **본문/설명** | 13~14px | 400 | line-height 1.5~1.6 |
| **힌트/메타** | 10~12px | 400 | 날짜, 부연설명 |

### 1-4. 인터랙션 & 애니메이션

| 패턴 | 구체적 수치 | 근거 앱 |
|------|------------|---------|
| **진행 바 트랜지션** | 700ms ease-out | Headspace, Calm |
| **카드 탭 피드백** | scale(0.97) 50ms | Reflectly, Daylio |
| **완료 애니메이션** | 체크 + 색상 전환 200ms | Headspace |
| **슬라이더 드래그** | 즉각 반응, 0 딜레이 | Reflectly |
| **페이지 전환** | fade 150ms or slide-up 200ms | Calm, Headspace |

### 1-5. 카드 컴포넌트

| 속성 | 값 | 이유 |
|------|-----|------|
| **border-radius** | 16~20px (rounded-2xl) | "친근함과 완성도" — 업계 표준 |
| **padding** | 16~20px (p-4 ~ p-5) | 내부 요소 숨쉬기 |
| **border** | 1px, 포인트컬러 10~15% 투명도 | 배경과의 분리, 과하지 않게 |
| **shadow** | 없거나 매우 미묘 | 다크모드에서 shadow 효과 약함 → border로 대체 |
| **gap between cards** | 12~16px (gap-3 ~ gap-4) | 섹션 호흡 |

## 2. 현재 베일러 코드베이스 분석

현재 구현된 세 모드:

| 파일 | 특징 | 문제점 |
|------|------|--------|
| `VentPage.tsx` (오리지널) | 대화 중심, amber 컬러, 자유형 | 시스템모드와 명확히 다름 — OK |
| `ClearHome.tsx` (클리어) | 대시보드, `#111318` 배경, sky-400+mint | 이미 구현됨. 시스템모드의 출발점 |
| `RoutineHome.tsx` (루틴) | 스트릭 링 + 7일 도트, amber | 독립 모드로 충분히 완결됨 |

**핵심 발견**: `ClearHome.tsx`가 이미 대시보드 구조를 가지고 있지만,  
리서치에서 도출한 패턴 — Adaptive 홈, 슬라이더 체크인, AI 인사이트, Daily Challenge — 이 아직 없다.

현재 ClearHome의 구조:
```
헤더 (닉네임 + 날짜)
↓
지표 3개 카드 행 (이번 주 기록 / 스트릭 / V-Score)
↓
오늘의 할 것 카드 (체크리스트)
↓
이번 주 세션 흐름 바차트
↓
V-Axis 스코어 프로그레스 바
```

**부족한 것**:
- 체크인 입력 UI (슬라이더 or 탭 선택) → 지금은 Vent 탭으로 이동하라는 링크만
- 단일 종합 점수 ("관계 건강도") — 지금은 V-Score라는 이름의 평균값만 있음
- Daily Challenge / AI 인사이트 섹션
- Adaptive 조건 (감정 점수에 따른 화면 변화)

## 3. 시스템모드 정의

### 이름

**"클리어" (Clear)** — 기존 가제 유지.  
이유: 이미 코드에 `ClearHome.tsx`, 디자인 토큰(`clear blue`)으로 일관되게 쓰임. 변경 비용이 높음.

### 한 줄 정의

> "지금 내 상태가 숫자로 보이고, 다음에 뭘 하면 되는지 자동으로 나온다."

### 타깃 유저 (리서치 기반)

| 페르소나 | 설명 |
|---------|------|
| **안정기 진입형** | 오리지널 2~4주 후 감정 기복이 줄고 "변화를 확인하고 싶은" 시점 |
| **성과 확인형** | "내가 얼마나 나아졌는가"를 데이터로 보고 싶음 |
| **구조 선호형** | 자유형 대화보다 "오늘 할 것"이 명확하게 나오는 것을 좋아함 |

### 오리지널 모드와의 차이

| 차원 | 오리지널 | 클리어 |
|------|---------|--------|
| 홈의 역할 | 엠버와 대화 시작 | 현재 상태 요약 + 다음 행동 제안 |
| 데이터 표현 | 감성 문장 ("3주째 월요일에 비슷한 감정이 나와") | 수치 + 그래프 |
| 입력 방식 | 자유 텍스트 / 음성 | 슬라이더 + 탭 선택 (2-tap) |
| 리텐션 장치 | 엠버 관계 언어 ("우리 얘기한 지 7일 됐어") | 주간 목표 바 + AI 인사이트 |
| 컬러 | amber-gold, warm dark | clear-blue, cool dark |

## 4. 프론트엔드 디자인 스펙

### 4-1. 컬러 토큰 (기존 ClearHome 기반 확정)

```
배경:           #0D1117  (현재 ClearHome 배경)
서피스:         #111318  (카드 배경)
서피스 2:       #161C24  (중첩 카드)
포인트:         #4AAEFF  (clear blue — sky-400)
성공/완료:      #34C48B  (mint — emerald)
경고:           #F59E0B  (amber — 낮은 점수)
보라:           #A78BFA  (V-Score 포인트)
텍스트 주:      #F1F5F9  (slate-100)
텍스트 보:      #94A3B8  (slate-400)
텍스트 힌트:    #475569  (slate-600)
경계:           포인트컬러 + /20 투명도
```

### 4-2. 타이포그래피

```
섹션 레이블:    11px, 400, letter-spacing 0.2em, uppercase, 텍스트힌트색
카드 제목:      15px, 600, 텍스트주색
핵심 수치:      28px, 700, tabular-nums, 포인트색
본문:           13px, 400, line-height 1.55, 텍스트보색
힌트:           11px, 400, 텍스트힌트색
```

### 4-3. 카드 컴포넌트 규칙

```
border-radius:  16px (rounded-2xl)
padding:        20px (p-5)
border:         1px solid [포인트색]/20
background:     #111318
gap:            12px (gap-3)
```

### 4-4. 애니메이션

```
진행 바:        transition: width 700ms ease-out
슬라이더:       즉각 반응
체크 완료:      scale 1→1.1→1 over 200ms, color transition 200ms
페이지 진입:    opacity 0→1 over 150ms
숫자 카운트업:  없음 (렌더링 복잡도 증가 대비 효과 낮음)
```

## 5. 홈 화면 상세 기획

### 5-1. 전체 구조

```
┌─────────────────────────────────────┐
│  ZONE A: 헤더 (고정 없음, 스크롤됨)  │
├─────────────────────────────────────┤
│  ZONE B: 핵심 상태 카드 [최우선]    │
├─────────────────────────────────────┤
│  ZONE C: 오늘 체크인               │
├─────────────────────────────────────┤
│  ZONE D: 이번 주 스냅샷            │
├─────────────────────────────────────┤
│  ZONE E: 오늘 추천 (챌린지 or 콘텐츠)│
├─────────────────────────────────────┤
│  ZONE F: AI 인사이트 (주 1회)       │
└─────────────────────────────────────┘
         ↕ 스크롤
```

### 5-2. ZONE A — 헤더

```
[모드 레이블] CLEAR           [설정 아이콘]
[날짜] 4월 14일 월요일
```

- 모드 레이블: 11px uppercase, slate-500
- 날짜: 13px, slate-400

### 5-3. ZONE B — 핵심 상태 카드

**"관계 건강도"** — 단일 종합 점수 (Calm의 Weekly Progress Goal 패턴)

```
┌─────────────────────────────────────┐
│  관계 건강도                 이번 주 │
│                                     │
│        [●●●●○]  78점               │
│  ─────────────────────────── ██░░  │
│                    목표까지 +3점    │
└─────────────────────────────────────┘
```

- 점수: 28px, bold, #4AAEFF
- 5-dot 표시 (●●●●○): 직관적 레벨 인식
- 주간 목표 진행 바: 얇은 1.5px 높이
- 점수 변화 방향 힌트: "지난주 +2점" (slate-400, 12px)

**산출 방식** (DB 쿼리 없이 프론트 계산):
```
관계 건강도 = (
  최근 7일 체크인 빈도점수 (0~40) +
  axis_scores 평균 (0~40) +
  스트릭 보너스 (0~20)
) / 100 * 100
```

### 5-4. ZONE C — 오늘 체크인

오늘 체크인이 **없을 때** (기본 상태):

```
┌─────────────────────────────────────┐
│  오늘 상태                          │
│                                     │
│  ○────────────●────────○           │
│  힘들어       보통      좋아        │
│                                     │
│  [관계]  [일]  [운동]  [혼자]  [+] │
│                                     │
│              [기록하기]             │
└─────────────────────────────────────┘
```

- 슬라이더: 1~10 범위, 드래그. 기본값 5 (중앙)
- 활동 탭: 4개 고정 + 커스텀 1개. 탭 = 토글 선택
- CTA: `bg-sky-500`, 텍스트 "기록하기"
- 소요 시간 표시: "약 10초" (마찰 제거)

오늘 체크인이 **있을 때**:

```
┌─────────────────────────────────────┐
│  오늘 상태     ✓ 완료              │
│  슬라이더 값: 7  · 활동: 운동, 관계 │
└─────────────────────────────────────┘
```

- 압축된 요약 카드로 전환
- 수정 가능 (탭하면 다시 열림)

### 5-5. ZONE D — 이번 주 스냅샷

두 가지 시각화를 한 카드에:

```
┌─────────────────────────────────────┐
│  이번 주                            │
│                                     │
│  [감정 7일 바 차트]                 │
│   월 화 수 목 금 토 일              │
│   ▄  █  ▃  █  ▄  ░  ▄             │
│                                     │
│  📊 운동한 날 기분 평균 +1.2점      │  ← Daylio 상관관계
└─────────────────────────────────────┘
```

- 바 차트: 현재 `ClearHome.tsx`의 `WeekBar` 컴포넌트 재사용
- 인사이트 1줄: 활동-감정 상관관계 중 가장 두드러진 것 1개만

### 5-6. ZONE E — 오늘 추천

**Daily Challenge** (Reflectly 패턴):

```
┌─────────────────────────────────────┐
│  오늘의 챌린지              선택    │
│  ─────────────────────────────────  │
│  "가까운 사람에게 먼저 연락해보기"  │
│                           [해봤어]  │
└─────────────────────────────────────┘
```

- 챌린지 목록: 정적 배열 (DB 연동 없이)
- 오늘 날짜 기반 순환 (dayOfYear % 챌린지수)
- [해봤어] 버튼 탭 → 체크 애니메이션 + localStorage 저장
- [선택] 버튼 → 다음 챌린지로 교체

### 5-7. ZONE F — AI 인사이트 (주 1회)

```
┌─────────────────────────────────────┐
│  이번 주 패턴                ✦ AI  │
│  ─────────────────────────────────  │
│  "운동한 날 감정 점수가 평균보다    │
│   1.4점 높게 나타났어요.            │
│   이번 주 3번 기록했네요."          │
└─────────────────────────────────────┘
```

- 표시 조건: 이번 주 체크인 3회 이상 + 마지막 표시로부터 7일 경과
- 생성 방식: 규칙 기반 (DB 추가 쿼리 없이 현재 주간 데이터로 계산)
- 없을 때: ZONE F 전체 숨김

## 6. 탭별 시스템모드 적용

### 6-1. Dig 탭 (패턴 탐색)

**오리지널**: 엠버가 발견한 것을 대화 카드로 서술  
**클리어**: 데이터 시각화 중심

```
[감정 트렌드] 탭 | [활동 분석] 탭

감정 트렌드 탭:
- 30일 감정 곡선 (꺾은선 그래프)
- 최고점/최저점 날짜 표시

활동 분석 탭:
- 활동별 평균 감정 점수 바 차트
- "운동 ▓▓▓▓▓ 7.2"
- "혼자   ▓▓▓░░ 5.8"
```

### 6-2. Get 탭 (콘텐츠)

**오리지널**: 엠버가 1가지 큐레이션  
**클리어**: 코스 카드 리스트 (Headspace 구조)

```
[진행 중 코스 카드] — 가장 위
  "관계 개선 4주 코스 · 2/12 세션"
  [계속하기]

[추천 코스 카드]
  "감정 표현 입문 · 3세션"
  "갈등 후 회복 · 4세션"
```

### 6-3. Set 탭 (목표)

**오리지널**: 엠버가 Vent 기반으로 목표 제안  
**클리어**: 목표 진행도 대시보드

```
[활성 목표 카드]
  "이번 달 체크인 20회"
  ████████░░  16/20

[완료된 목표]
  ✓ "일주일 연속 기록"  완료 3.14
```

### 6-4. Me 탭

**오리지널**: "이번 달의 나" 서술 + 감정 히트맵 amber 농도  
**클리어**: 수치 대시보드 + 월간 캘린더 그리드

```
[이번 달 요약 수치]
  기록 18일  평균 6.8점  최고 9점

[월간 캘린더 히트맵]
  각 날짜를 점수 색상으로 채움
  빈 날: stone-800 / 1~4점: amber / 5~7점: sky / 8~10점: emerald
```
