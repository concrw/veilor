## §4. 프로덕트 설계

### 4-A. Consent Module — RI API 확장

기존 RI API(Relationship Intelligence API)에 consent module을 추가한다. 이 모듈은 유저의 합의 상태를 실시간으로 관리하며, 콘텐츠 접근 권한을 제어한다.

#### 4-A-1. 데이터 구조

```
User Profile
├── identity_verified: boolean
├── age_verified: boolean (3rd party — Jumio/Onfido)
├── consent_state: {
│     level_1_3: {
│       granted: true (기본 동의, 가입 시 자동),
│       timestamp: ISO datetime,
│       revocable: true
│     }
│     level_4: {
│       granted: boolean,
│       timestamp: ISO datetime,
│       method: "explicit_click" (레이어드 합의 3단계 완료),
│       comprehension_depth: L1 | L2 | L3 (유저가 도달한 설명 깊이)
│     }
│     level_5: {
│       granted: boolean,
│       timestamp: ISO datetime,
│       method: "double_opt_in + age_verify",
│       comprehension_depth: L1 | L2 | L3
│     }
│   }
├── consent_history: [ ...all changes with timestamps ] (append-only, 불변)
├── meta_consent: boolean (합의 행위 데이터의 연구 활용 동의)
├── disclosure_depth_log: {
│     framework_results: L1 | L2 | L3 (진단 결과 탐색 깊이),
│     content_consumption: { topic_id: depth_reached } (콘텐츠별 탐색 깊이)
│   }
└── data_deletion_requested: boolean
```

#### 4-A-2. 핵심 로직

**게이트키핑:** 유저가 Lv.4 이상 콘텐츠에 접근할 때 consent_state를 실시간 체크한다. 미동의 시 콘텐츠 차단 + 레이어드 합의 플로우 트리거. 이때 "차단" 화면은 벽이 아니라 문으로 설계 — "여기부터는 더 깊은 영역입니다. 들어가시겠습니까?"

**불변 로그:** consent_history는 append-only 구조다. 수정 및 삭제가 불가능하다. 유저가 언제, 어떤 정보를 확인하고, 어떤 깊이까지 탐색한 뒤, 어떤 선택을 했는지가 모두 기록된다. 이 로그는 법적 감사 시 증거로 활용된다.

**즉시 철회:** 유저가 동의를 철회하면 해당 수위의 콘텐츠 접근이 즉각 차단된다. 관련 데이터에 대한 소거 프로세스가 자동 트리거된다. 철회 이유를 묻지 않는다.

**메타 합의:** 유저의 합의 행위 자체(어떤 수위를 선택했는지, 어떤 설명 깊이까지 탐색했는지)를 연구 데이터로 활용하려면 별도의 동의가 필요하다. consent_state 내의 `meta_consent` 필드로 관리한다.

#### 4-A-3. 인터랙티브 합의 UX — 레이어드 마이크로 경험

단순 체크박스를 레이어드 인터랙티브 경험으로 대체한다. 프로그레시브 디스클로저(progressive disclosure) 패턴 적용.

**Lv.4 합의 플로우 — 3단계 스와이프 (5초 이내):**

화면 1 (L1): 한 문장 주제 안내 + 스와이프 유도

화면 2 (L2): 핵심 개념 재정의 (경고 아닌 초대 톤)

화면 3: "준비되셨습니까?" + "네, 들어갈게요" / "아직은 아니에요" (페널티 없음)

**설계 원칙:**

- 3회 이내 인터랙션 (5초 이내)
- 텍스트 벽이 아닌 한 문장씩
- "아직은 아니에요"에 페널티 없음 — 돌아갈 수 있고, 다시 올 수 있음
- 카피의 톤은 경고가 아니라 초대 — Leo Vargas(CMO)가 톤 관리

**Lv.5 합의 플로우 — 이중 인증 추가:**

위 3단계 + 연령 검증(3rd party ID verification) + 명시적 클릭 + 24시간 이내 철회 안내 팝업

### 4-B. 앱 내 레이어링 — 하이브리드 구조

**방향 A — 앱 내 텍스트 레이어링:** 아코디언 UI. 탭할수록 L1→L2→L3로 펼쳐짐.

**방향 B — 콘텐츠 연결형:** L1 앱 내 텍스트 / L2 영상·팟캐스트 링크 / L3 풀 아티클 + 인터랙티브 코드토크 세션.

**하이브리드 채택 이유:** A만 쓰면 텍스트에 갇힌다. B만 쓰면 앱 밖으로 유저가 빠진다. A+B 하이브리드는 체류 시간을 늘리고 데이터 축적 플라이휠과 연결된다.

### 4-C. 세그먼테이션 자동화

| 유저 행동 | 세그먼트 | 활용 |
| --- | --- | --- |
| L1에서 멈춤 | Casual — 가벼운 관심 | 숏폼 콘텐츠 추천, Lv.1~2 |
| L2까지 탐색 | Engaged — 자기 이해 욕구 | 진단 도구 추천, 유튜브 시리즈 |
| L3까지 파고듦 | Deep — 변화 의지 | 코칭 추천, Lv.4~5 콘텐츠, 프리미엄 |

별도 설문 없이 유저의 자연스러운 행동에서 추출. RI API 매칭·콘텐츠 추천·AI 코치 톤 조절 전부 활용.

## §5. 법적 방어 구조

### 5-A. 규제 매핑

#### 5-A-1. GDPR (EU)

| GDPR 조항 | Veilrum 합의 프레임워크 대응 |
| --- | --- |
| Article 7 — 동의 조건 | 레이어드 합의 3단계 = "자유롭고 구체적이며 충분한 정보에 입각한 동의" |
| Article 17 — 삭제권 | 데이터 삭제 원클릭 + 완전 소거 파이프라인 |
| Article 7(3) — 철회권 | 철회 원클릭, 철회 이유 미요구, 철회 후 불이익 없음 |
| Article 25 — 프라이버시 바이 디자인 | consent module이 프로덕트에 내장 (설계 단계부터) |

#### 5-A-2. CCPA (캘리포니아)

| CCPA 요건 | 대응 |
| --- | --- |
| 데이터 삭제권 | GDPR 대응과 동일 |
| 옵트아웃 | 수위별 합의 분리 — 각 수위를 독립적으로 옵트아웃 가능 |
| 비차별 | 합의 수위와 무관하게 기본 서비스 동일 |

#### 5-A-3. 콘텐츠 관련

| 법적 쟁점 | 대응 |
| --- | --- |
| Section 230 (미국) | 자체 플랫폼 내 유저 동의 기반 콘텐츠 → 플랫폼 면책 |
| COPPA (아동보호) | 미성년자 접근 원천 차단 + 연령 검증 |
| 음란물 규제 | Lv.5 콘텐츠는 학술/교육 맥락의 성적 내러티브 — 포르노가 아닌 연구 기반 분석 |

### 5-B. 증거 보존 체계

| 증거 항목 | 저장 위치 | 보존 기간 |
| --- | --- | --- |
| 합의 3단계 각 단계 확인 기록 | consent_history (append-only) | 계정 삭제 후 5년 |
| 유저가 확인한 설명 깊이 (L1/L2/L3) | disclosure_depth_log | 동일 |
| 연령 검증 결과 | 3rd party 인증 기록 | 동일 |
| 철회 기록 + 타임스탬프 | consent_history | 동일 |
| 메타 합의 상태 | consent_state | 동일 |

### 5-C. 실무 구현 우선순위

| 항목 | 구현 방법 | 우선순위 |
| --- | --- | --- |
| ------ | --------- | :---: |
| 연령 검증 시스템 | 3rd party ID verification (Jumio 또는 Onfido) | 🔴 최우선 |
| 동의 로그 타임스탬프 + 불변 저장 | consent_history DB (append-only, 암호화) | 🔴 최우선 |
| 철회 원클릭 + 데이터 소거 | GDPR 준수 삭제 파이프라인 | 🔴 최우선 |
| 수위별 동의 분리 | Lv.1~3 기본 / Lv.4 명시 / Lv.5 이중 인증 | 🔴 최우선 |
| 인터랙티브 합의 UX | 레이어드 스와이프 플로우 (§4-A-3) | 🟡 |
| 다크패턴 감사 | 분기별 외부 UX 감사 | 🟡 |
| 메타 합의 별도 수집 | meta_consent 필드 + 별도 동의 화면 | 🟡 |
| A/B 테스트 — 합의 플로우 최적화 | 스와이프 횟수, 카피 톤, 전환율 비교 | 🟢 런칭 후 |

### 5-D. 외부 공격 시나리오별 대응

| 공격 시나리오 | 대응 |
| --- | --- |
| 종교 단체: "이 콘텐츠가 도덕적이냐" | "우리는 도덕을 판단하지 않습니다. 합의를 확인합니다." |
| 보수 미디어: "도덕적 상대주의" | "합의의 조건은 절대적입니다. 정보 대칭, 자유 의지, 철회 가능성." |
| 규제 기관: "유해 콘텐츠 유통" | 합의 로그 + 연령 검증 + 레이어드 설명 기록 제출 |
| 유저 소송: "이런 내용에 노출되었다" | 3단계 합의 기록 + 타임스탬프 + 철회 가능 안내 기록 제출 |
| 경쟁사/언론: "그냥 성인 콘텐츠 아니냐" | "관계 심리학 연구 기반 분석 콘텐츠. 데이터와 프레임워크가 있다." |
