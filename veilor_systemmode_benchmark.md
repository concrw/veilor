## 1. DAYLIO — 극저마찰 감정 체크인

### 홈 화면 구조

- **첫 진입 CTA**: 5개 감정 이모지 (Rad → Good → Meh → Bad → Awful) 전면 노출
- **레이아웃**: 이모지 그리드가 화면의 60% 이상 차지. 텍스트는 보조
- **첫 행동 유도**: 화면 열자마자 탭 가능한 상태. 스크롤 없이 완결
- **달력 뷰**: 과거 기록이 색상 픽셀 그리드로 표시되어 홈 하단에 항상 노출

### 핵심 인터랙션 패턴

- **3단계 체크인**: 감정 선택(탭) → 활동 선택(탭) → 노트 입력(선택). 노트 없이도 완료 가능
- **텍스트 입력 강제 없음**: 타이핑 없이 로그 완료 → "귀찮아서 안 하는" 이탈률 최소화
- **커스터마이즈**: 이모지·활동 아이콘 자유 편집 → 내 언어로 내 감정을 기록
- **즉각 반응**: 탭 → 애니메이션 → 완료 메시지. 지연 없음

### 진척/데이터 시각화

| 시각화 | 방식 | 인사이트 제공 |
|--------|------|-------------|
| **Calendar View** | 일별 감정 색상 픽셀 그리드 | "내 한 달이 한눈에" |
| **Mood Chart** | 주/월별 감정 트렌드 곡선 | 트렌드 방향 인식 |
| **Activity Correlation** | 특정 활동 → 감정 영향도 수치 | "운동하면 기분 좋아지는 것 같다" 데이터 확인 |
| **Year in Pixels** | 연단위 감정 색상 타일 | 장기 패턴 시각화 |
| **Streak 카운터** | 홈 상단 연속 기록 일수 | 손실 회피 심리 자극 |

### 베일러 시스템모드 적용 포인트

- ✅ **극저마찰 감정 입력**: 슬라이더 or 탭-3개 이내로 오늘 체크인 완료
- ✅ **활동-감정 상관관계**: "이 행동이 내 관계/감정에 미친 영향" 상관도 표시
- ✅ **Calendar 히트맵**: 베일러 감정 히트맵을 플로우모드(amber 농도)에서 시스템모드(숫자+컬러 스케일)로 전환

## 2. HEADSPACE — 코스 기반 선형 진행

### 홈 화면 구조

- **첫 진입 CTA**: 진행 중인 코스 카드 (제목 + 회차 + "계속하기" 버튼) 상단 고정
- **레이아웃**: 카드 1 (현재 코스) → 카드 2~N (추천 코스) 수직 스크롤
- **탭 구조**: Home | My Stats | My Journey (3개)
- **홈의 역할**: 선택지 제시가 아닌 "지금 해야 할 것" 하나만 강조

### 핵심 인터랙션 패턴

- **선형 진행**: 1회차 → 2회차 → 3회차. 이전 완료 표시(체크). 다음 회차만 활성화
- **1-tap 시작**: 카드 탭 → 즉시 재생. 선택 피로 없음
- **세션 내 진행 바**: 하단 고정 프로그레스 바로 현재 위치 실시간 표시
- **회차 완료 피드백**: 애니메이션 + 한 마디 메시지 (심리적 성취감)

### 진척/데이터 시각화

| 시각화 | 방식 | 강조 요소 |
|--------|------|----------|
| **Run Streak** | 연속 명상 일수 (불꽃 이모지) | 손실 회피 |
| **Total Sessions** | 누적 세션 수 | 자산화 |
| **Mindful Minutes** | 총 명상 시간 (분) | 투자 인식 |
| **My Journey** | 세션 타임라인 (날짜별 리스트) | "내가 걸어온 길" |
| **Achievement Badges** | 7일/21일/365일 마일스톤 배지 | 도달 목표 제시 |

### 베일러 시스템모드 적용 포인트

- ✅ **코스 구조**: 관계 개선 여정을 회차 기반 코스로 제공 → "지금 어느 단계인가" 명확화
- ✅ **선형 진행 UX**: 다음에 뭘 해야 하는지 자동 제안 (선택 피로 0)
- ✅ **My Journey**: 베일러의 관계·감정 개선 타임라인을 날짜별로 시각화

## 3. BETTERUP — Adaptive 대시보드 + 코칭 연결

### 홈 화면 구조

- **레이아웃**: 오늘의 상태에 따라 바뀌는 Adaptive Dashboard
  - 코치 추천 카드 (프로필+추천 이유)
  - 진행 중인 목표 진행도 (%)
  - 다음 세션 일정 카드
- **탭 구조**: Home | Coaching | Goals | Insights (4개)
- **홈의 역할**: "오늘 가장 중요한 것 1가지"를 맥락에 맞게 노출

### 핵심 인터랙션 패턴

- **목표 설정 → 진행 → 코치 리뷰** 3단계 루프
- **GROW 모델 구조**: Goal → Reality → Options → Will (내러티브형 목표 관리)
- **코치 메시지**: 세션 외에도 앱 내 비동기 메시지로 지속 상호작용
- **세션 예약 강제성**: 정기 예약 = 앱 이탈 방지의 구조적 장치

### 진척/데이터 시각화

| 시각화 | 방식 | 역할 |
|--------|------|------|
| **Goal Progress** | 목표별 진행도 % + 마지막 업데이트 | "내 목표가 얼마나 됐나" |
| **Insights Panel** | 코치 제안 콘텐츠 카드 (오른쪽) | 맥락 기반 추천 |
| **Session History** | 코칭 세션 노트 + 주요 결정 사항 | 성장 회고 |
| **Team Analytics** (B2B) | 팀 평균 멘탈 지표 대시보드 | 조직 차원 모니터링 |

### 베일러 시스템모드 적용 포인트

- ✅ **Adaptive 홈**: 사용자의 현재 상태(감정 점수, 최근 기록)에 따라 다른 화면 구성
- ✅ **목표 → 진행도 루프**: 관계 개선 목표를 수치로 추적하는 구조
- ✅ **콘텐츠 추천**: "지금 이 상태에 필요한 것" 기반 1개 추천 카드 (Get 탭과 연동)

## 4. REFLECTLY — AI 인사이트 + 감정 슬라이더

### 홈 화면 구조

- **레이아웃**: 일일 체크인 카드 (감정 슬라이더 포함) + 지난 기록 피드
- **시각 계층**: 슬라이더 CTA → 이번 주 그래프 → Daily Challenge → 과거 피드
- **AI 존재감**: 홈 하단에 "지난주 대비 감정이 X% 개선됐어요" 텍스트 인사이트 노출

### 핵심 인터랙션 패턴

- **감정 슬라이더**: 1~10 드래그로 기분 입력 → 노트 입력 → AI 인사이트 생성
- **프롬프트 기반 저널**: "오늘 뭐가 제일 좋았어?" 같은 가이드 질문 제시 (빈 화면 공포 제거)
- **Daily Challenge**: 매일 달라지는 소규모 과제 (칭찬 3개 쓰기 / 오늘 배운 것 1줄 등)
- **AI 피드백 타이밍**: 기록 직후 즉각 생성 → 기록 동기 강화

### 진척/데이터 시각화

| 시각화 | 방식 | 특징 |
|--------|------|------|
| **Mood Graph** | 주/월/연도별 감정 곡선 | 트렌드 패턴 인식 |
| **Habit Impact** | 특정 활동-감정 상관관계 | "이게 영향을 미쳤구나" 자기발견 |
| **AI Weekly Insight** | 텍스트 요약 ("이번 주 당신은 X에 많이 반응했어요") | 해석 대행 |
| **Mood Triggers** | 감정 유발 요소 리스트 + 빈도 | 행동 변화 기반 제공 |

### 베일러 시스템모드 적용 포인트

- ✅ **슬라이더 + 탭 조합**: 플로우모드의 자유형 대화에서 구조형 입력으로 전환 시 마찰 최소화
- ✅ **AI 인사이트**: 베일러의 가장 강력한 리텐션 드라이버가 될 수 있음. "이번 달 당신의 감정 패턴은..."
- ✅ **Daily Challenge**: 관계 개선을 위한 소규모 행동 과제 (오늘 5분 대화 해보기 등)

## 5. WOEBOT — 대화형 CBT 챗봇

> ⚠️ 2025년 6월 30일 B2C 서비스 종료, B2B로 전환. 설계 참고용으로만 활용.

### 홈 화면 구조

- **레이아웃**: 채팅 창이 전부. 메시지가 홈이자 서비스
- **탭 구조**: Chat | Mood | Resources | Journal (4개)
- **첫 CTA**: "오늘 어떤 기분이야?" Woebot의 자연스러운 인사로 시작

### 핵심 인터랙션 패턴

- **선택지 기반 대화**: 자유 텍스트 + 다중선택지 병행 (AI 이해도 향상 + 마찰 감소)
- **CBT 흐름**: 기분 수집 → 긍정(심화학습) / 부정(즉각 대처 도구 제시) 분기
- **규칙 기반 공감**: 심리학자 작성 스크립트 → 예측 가능한 신뢰감
- **비강압 설계**: 제안하되 강요하지 않음 → 번아웃 없음

### 진척/데이터 시각화

| 시각화 | 방식 |
|--------|------|
| **Mood Chart** | 시간대별 기분 변화 그래프 |
| **Check-in History** | 과거 대화 로그 |
| **Insight Messages** | 심리교육 콘텐츠 (짧은 텍스트) |

### 베일러 시스템모드 적용 포인트

- ✅ **규칙 기반 대화 흐름**: 감정 상태별 분기 (안정 → 심화 / 위기 → 즉각 개입)
- ✅ **비강압 선택지 설계**: 항상 거절 옵션 제공 (베일러 플로우모드 원칙 유지)
- ⚠️ **서비스 종료 교훈**: B2C 멘탈헬스 챗봇만으로는 비즈니스 지속 어려움 → 코칭/콘텐츠 조합 필요

## 6. CALM — 강력한 온보딩 + ML 콘텐츠 추천

### 홈 화면 구조

- **온보딩**: 목표 선택 퀴즈 (수면/스트레스/명상 등) → 개인화된 홈 즉시 구성
- **메인 탭 구조**: Meditate (중앙·최대) | Music | Sleep
- **홈 구성**: Sleep Readiness Bar + 오늘 추천 명상 + 저녁 루틴 체크리스트

### 핵심 인터랙션 패턴

- **1-tap 시작**: 추천 콘텐츠 탭 1회 = 즉시 재생 (선택 피로 0)
- **배경음 연속성**: 콘텐츠 탐색 중에도 자연 소리 재생 → 분위기 유지
- **Sleep Readiness**: "오늘 수면 준비도 75%" 단일 수치로 상태 요약
- **ML 추천**: Amazon Personalize 기반 → 스크롤 없이 맞는 콘텐츠 노출

### 진척/데이터 시각화

| 시각화 | 방식 | 특징 |
|--------|------|------|
| **Streak + Stats** | 연속일 + 총 세션 수 + 누적 시간 | 프로필 대시보드 |
| **Monthly Calendar** | 월별 명상 기록 | 달력 히스토리 |
| **Weekly Progress Goal** | 주간 목표 진행 바 (리셋됨) | 우상향 느낌 |
| **Sleep Readiness** | 오늘의 준비도 % | 단일 핵심 지표 |

### 베일러 시스템모드 적용 포인트

- ✅ **온보딩 퀴즈**: 첫 모드 전환 시 "지금 가장 집중하고 싶은 것은?" 3가지 선택 → 대시보드 개인화
- ✅ **단일 핵심 지표**: "오늘 관계 건강도 73%" 같은 종합 지표 하나 (Sleep Readiness 벤치마크)
- ✅ **ML/AI 추천**: 현재 상태에 맞는 콘텐츠 1개 자동 추천 (Get 탭 자동화)

## 8. VEILOR 시스템모드 UX 구조 제안

### 8-1. 모드 정체성 재정의

**이름 후보**
- `시스템모드` (현재 가제) — 명확하지만 차갑다
- `대시` (Dashboard의 줄임) — 친근하고 기억하기 쉬움
- `클리어` (기존 가제) — 직관적 의미 전달

**한 줄 정의**: "지금 어디쯤 와 있는지 한눈에. 다음에 뭘 하면 되는지 자동으로."

**타깃 페르소나**
- 플로우모드 2~4주 사용 후 안정기 진입 유저
- 감정 기록보다 "변화 확인"이 더 중요해진 시점의 유저
- 구조화된 개선 과정을 원하는 유저

### 8-2. 홈 화면 구조 제안

```
┌─────────────────────────────────┐
│  [Header]                       │
│  오늘 ○월 ○일 · 안녕, 이름      │
```

### 8-3. 탭별 UX 차이 (플로우모드 대비)

| 탭 | 플로우모드 (오리지널) | 시스템모드 |
|----|------------------|-----------|
| **홈** | 엠버 인사 + 자유대화 | 대시보드 (지표+체크인+추천) |
| **Dig** | "자주 느끼는 것" 카드 서술형 | 감정 트렌드 그래프 + 활동 상관관계 표 |
| **Get** | 엠버가 1가지 큐레이션 | 상태 기반 코스 카드 (Headspace 구조) |
| **Set** | 엠버가 Vent 기반 제안 | 목표 % 진행도 (BetterUp Goal 구조) |
| **Me** | "이번 달의 나" 서술 | 수치 대시보드 + Calendar 히트맵 |

### 8-4. 핵심 인터랙션 패턴 (검증된 조합)

**일일 체크인 플로우** (Daylio + Reflectly 결합)
```
1. 감정 슬라이더 (1~10) — 3초 완료 가능
2. 활동 탭 선택 (최대 3개, 원탭) — 선택 안 해도 OK
3. 한 줄 노트 (선택) → [저장]
4. 즉각 피드백: "오늘로 12일째. 지난주보다 +1점"
```

**코스 진행** (Headspace 구조)
```
관계 개선 로드맵 — 4주 코스
  └─ 1주차: 자기 감정 인식 (3개 세션)
  └─ 2주차: 감정 표현 언어 (3개 세션)
  └─ 3주차: 대화 시작하기 (3개 세션)
  └─ 4주차: 갈등 후 회복 (3개 세션)

각 세션: 5분 내 완료 가능. 완료 시 체크 + 애니메이션.
```

**Adaptive 홈** (BetterUp 벤치마크)
```
감정 점수 높은 날 → 심화 콘텐츠 카드 노출
감정 점수 낮은 날 → "오늘은 쉬어도 괜찮아" + 소규모 챌린지
3일 이상 기록 없을 때 → 재진입 넛지 카드
```

### 8-5. 데이터 시각화 원칙

1. **단일 핵심 지표 1개** (관계 건강도 점수) — Calm Sleep Readiness 벤치마크
   - 계산식: 최근 7일 감정 평균 + 기록 빈도 + 코스 진행도의 가중 합산

2. **감정-행동 상관관계** — Daylio 벤치마크
   - "친구와 대화한 날 vs 혼자 있던 날"의 감정 점수 차이 시각화

3. **주간 곡선 소형 그래프** — Reflectly 벤치마크
   - 7일 감정 트렌드. 수치보다 방향(우상향/하락/안정)이 중요

4. **코스 진행 타임라인** — Headspace 벤치마크
   - 완료한 세션에 체크. "여기까지 왔어요" 누적 자산화

5. **월간 캘린더 히트맵** — Daylio 벤치마크
   - 감정 점수를 색 농도로 표시. 1달이 한 화면에 요약

### 8-6. 리텐션 메커니즘

| 메커니즘 | 참고 앱 | 베일러 적용 방식 | 주의사항 |
|---------|---------|----------------|---------|
| 누적 기록 강조 | Daylio | "12일째 기록 중" (연속보다 누적 강조) | 스트릭 끊김 죄책감 금지 |
| AI 인사이트 | Reflectly | 주 1회 패턴 분석 텍스트 | 과도한 분석 피로도 주의 |
| Daily Challenge | Reflectly | 관계 개선 소규모 행동 과제 | 강제성 없음, 선택형 |
| 코스 완료 배지 | Headspace | 4주 코스 완료 시 | 중간 이탈 방지가 더 중요 |
| 시간대별 알림 | Calm | 가장 많이 사용하는 시간 1개 | 최대 1일 1회 |
| Adaptive 홈 | BetterUp | 상태별 다른 화면 구성 | 기술 복잡도 고려 |

### 8-7. 설계 금기사항

1. **스트릭 끊김 = 실패** 느낌 금지 (멘탈헬스 앱의 역설)
   → 연속이 아닌 "누적량"으로 표시. "총 12일 기록" > "12일 연속"

2. **알림 과잉**: 일 1회 이상 금지. 사용자 활동 시간대 1개만
   → 연구: 일 3회 이상 = 삭제율 34% 증가

3. **데이터 오버로드**: 지표 5개 이상 동시 노출 금지
   → 핵심 지표 1개 + 보조 2개 이하

4. **빈 화면 공포**: 첫 체크인 전 화면에 "아직 기록이 없습니다" 금지
   → 첫 진입 = 체크인 CTA 즉시 노출

5. **강제 목표 입력**: 베일러는 유저가 목표를 직접 입력하지 않음
   → AI/코스가 현재 기록 기반으로 제안 (BetterUp의 코치 추천 방식)
