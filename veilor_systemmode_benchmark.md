## 1. DAYLIO

> 출처: [daylio.net/faq/activity-and-mood-statistics](https://daylio.net/faq/activity-and-mood-statistics/) · [appsreviewnest.com](https://appsreviewnest.com/app-review/daylio-app-review-a-helpful-tool-for-mood-tracking-and-journaling/) · [moodtrackers.org](https://moodtrackers.org/daylio-app-review/)

### 홈 화면 구조

- 상단에 **"+" 버튼** 하나가 중심. 탭하면 "오늘의 기분 기록" or "중요한 날짜 추가" 두 옵션
- 홈 화면 본체는 **날짜별 기록 리스트** (이모지 + 활동 태그 + 짧은 메모 형태)
- 하단 탭 네비게이션: **Journal | Calendar | Stats | Settings**
- 기록이 없는 날은 빈 줄, 기록이 있으면 이모지+컬러로 시각적 밀도 형성

### 핵심 인터랙션 패턴

- **2-tap 체크인**: 기분 선택(탭) → 활동 선택(탭). 이것만으로 엔트리 완료
- 텍스트 노트, 사진, 음성 첨부는 **모두 선택사항** — 최소 마찰이 핵심 철학
- 기분 이모지 5개는 커스터마이즈 가능 (색상 + 이모지 직접 선택)
- 활동 태그: 감정·건강·취미·사교·자기개발·생산성 카테고리, 커스터마이즈 가능
- **템플릿 기능**: 정해진 질문에 답하는 구조화 모드 (선택적)

### 진척/데이터 시각화

Stats 탭에서 제공하는 시각화 (공식 Knowledge Base 확인):

| 차트 | 방식 | 인사이트 |
|------|------|---------|
| **Mood Chart** | 주/월/연도별 기분 트렌드 꺾은선 | 감정 변화 방향 파악 |
| **Calendar View** | 일별 기분 색상 픽셀 그리드 | 한 달을 한눈에 |
| **Frequency** | 선택 기간 내 반복 횟수 vs 이전 기간 비교 | 상대적 변화 |
| **Mood Count** | 특정 활동의 기분별 분포 및 비율 | 활동-감정 패턴 |
| **Influence on Mood** | 활동 포함/미포함 시 기분 비교 (신뢰도 수준 표시) | 상관관계 근거 |
| **Occurrence During Week** | 요일별 활동/기분 빈도 막대 그래프 | 루틴 패턴 |
| **Related Activities** | 특정 기분과 함께 기록된 상위 활동 목록 | 트리거 발견 |

**"Influence on Mood"의 4가지 비교 지표** (검증된 기능):
1. 활동 있는 날 vs 없는 날 기분 평균
2. 활동 전날 기분
3. 같은 날 전체 기분
4. 활동 다음날 기분

→ 사용자가 "이 활동이 내 기분에 실제로 영향을 주는가"를 데이터로 확인 가능

### 베일러 시스템모드 적용 포인트

- ✅ **2-tap 체크인 구조**: 슬라이더 or 탭 선택으로 오늘 감정 + 관계 상황 2단계 완료
- ✅ **활동-감정 상관관계**: "이 관계에서의 상호작용 → 내 감정 변화" 연결 시각화
- ✅ **요일별 패턴**: "매주 월요일 감정 점수가 낮다" 같은 패턴 발견 도구
- ⚠️ 커스터마이즈는 강점이지만 복잡도 증가 → 베일러는 베일러 고유 언어 유지

## 2. HEADSPACE

> 출처: [ixd.prattsi.org — Design Critique: Headspace iOS App](https://ixd.prattsi.org/2026/02/design-critique-headspace-ios-app-2/) · [mobbin.com](https://mobbin.com/explore/screens/78be544a-d7ea-4596-9038-363769239d36) · [ngrow.ai — Engagement 32% increase study](https://www.ngrow.ai/blog/how-headspace-increased-engagement-by-32-with-strategic-push-notifications)

### 홈 화면 구조

- **현재 진행 중인 코스 카드** 상단 고정: 코스명 + 현재 회차 + "계속하기" 버튼
- 그 아래: 추천 콘텐츠 카드들 수직 스크롤
- 홈의 핵심 역할: "지금 해야 할 것 1개"를 명확히 노출
- Explore 탭 별도 존재 (콘텐츠 탐색). 홈과 탐색을 분리한 구조
- Pratt IXD 리뷰에서 지적: 초급 코스 이후 Explore 섹션이 시각적으로 복잡해짐 — 약점으로 인식됨

### 핵심 인터랙션 패턴

- **선형 진행 구조**: "명상은 단계적 여정" — 1회차 완료 → 2회차 활성화. 선택 피로 없음
- 시각적 배치 자체가 순서를 설명: 완료된 회차(체크 표시) → 현재(활성) → 이후(비활성)
- 세션 내 **하단 고정 진행 바**: 실시간 위치 피드백 → 불확실성 제거
- 세션 완료 시 **애니메이션 + 텍스트 피드백** (심리적 성취감)
- 1-tap 시작: 카드 탭 → 즉시 재생

### 진척/데이터 시각화

| 항목 | 방식 |
|------|------|
| **Run Streak** | 연속 명상 일수 (불꽃 아이콘) |
| **Total Sessions** | 누적 세션 수 |
| **Mindful Minutes** | 총 명상 시간 (분 단위) |
| **My Journey** | 세션 타임라인 (날짜별 완료 목록) |
| **Achievement Badges** | 7일·21일·365일 마일스톤 |

**비판적 인사이트** (Pratt IXD 리뷰에서):
> "스트릭 카운트는 내재적 동기('명상으로 더 나아지기')를 외재적 추적('숫자를 지키기')으로 변환시킨다. 장기적 자율성을 훼손할 수 있다."

### 베일러 시스템모드 적용 포인트

- ✅ **선형 코스 구조**: 관계 개선 여정을 회차 기반으로 제공 → "지금 어느 단계인가" 명확화
- ✅ **"현재 할 것 1개" 홈 구조**: 선택 피로 없이 진입점 제공
- ⚠️ **스트릭의 역설**: Headspace도 비판받는 패턴. 베일러는 연속 대신 누적 강조

## 3. BETTERUP

> 출처: [betterup.com/platform-releases/fall-2025](https://www.betterup.com/platform-releases/fall-2025) · [mentorcruise.com](https://mentorcruise.com/blog/an-honest-2024-betterup-review/) · [adriannagonzalez.com — Scalable UX](https://www.adriannagonzalez.com/betterup-multicoach)

### 홈 화면 구조

- **Adaptive 홈**: 사용자 상태에 따라 콘텐츠 구성이 다름
  - 세션 예약 없을 때: 코치 추천 카드 + "세션 잡기" CTA
  - 세션 예약 후: 다음 세션 일정 카드 + 준비 체크리스트
  - 세션 후: 실행 과제 카드 + 다음 목표 제안
- 2025 Fall 업데이트: **Coaching 탭에서 Goals 섹션이 더 두드러지게 재편**
- "다음 단계 안내"에 집중하는 구조 — 데이터 대시보드보다 행동 유도 우선

### 핵심 인터랙션 패턴

- 가입 후 **20분 인테이크 세션** → 코치 매칭 → 정기 예약
- **Human + AI 이중 구조** (2025): AI 코치가 일상 스킬 훈련, 인간 코치가 고난도 돌파구
  - Slack·Teams 통합: 앱 전환 없이 코칭 접근
  - 캘린더 통합: 중요한 미팅 전 맥락 기반 코칭
- **AI Readiness Score**: Insights 섹션에서 AI 코치와의 인터랙션 기반 진행도 표시
- 세션 중 **AI가 실시간 인사이트 캡처** → 세션 후 정리본 제공

### 진척/데이터 시각화

| 항목 | 방식 |
|------|------|
| **Goal Progress** | 목표별 진행도 % + 마지막 업데이트 시간 |
| **AI Readiness Score** | Insights 탭에서 수치로 표시 |
| **Unified Reporting** | 인간+AI 코칭 전체 참여도/영향도 통합 (조직 관리자용) |
| **Session History** | 과거 세션 노트 + 주요 결정 사항 |

**실제 사용자 리뷰** (MentorCruise):
- "직관적이고 접근하기 쉬운 인터페이스"
- "세션 후 구체적인 행동 계획 수립 지원"
- "가격이 큰 진입 장벽"
- "코치 품질 편차가 큼"

### 베일러 시스템모드 적용 포인트

- ✅ **Adaptive 홈**: 유저 상태에 따라 다른 화면 → 항상 "지금 가장 필요한 것" 1개 노출
- ✅ **Goal Progress**: 관계 개선 목표를 수치로 추적하는 구조
- ✅ **행동 계획 우선**: 데이터 보여주기보다 "다음에 뭘 해야 하는가" 중심
- ⚠️ 코치 의존 구조는 베일러 B2B 모드에 더 적합. B2C 시스템모드는 AI 대체

## 4. REFLECTLY

> 출처: [screensdesign.com/showcase/reflectly](https://screensdesign.com/showcase/reflectly-journal-ai-diary) · [aiapps.com](https://www.aiapps.com/blog/reflectly-app-review/) · [pageflows.com/ios/products/reflectly](https://pageflows.com/ios/products/reflectly/)

### 홈 화면 구조

- **메인 대시보드** (4분 7초 타임스탬프에서 확인): 일일 과제를 카드로 분리
  - 프롬프트 카드 / 챌린지 카드 / 체크인 카드 — 각각 독립 카드
- 홈 상단: 오늘 날짜 + 기분 슬라이더 CTA
- 홈 하단: 과거 저널 피드 (최신순 스크롤)
- AI 인사이트는 별도 섹션 or 저널 작성 직후 표시

### 핵심 인터랙션 패턴

- **감정 슬라이더**: 드래그로 기분 1~10 입력. "delightful slider and clear iconography" — screensdesign 확인
- **1분 7초 타임스탬프**: 슬라이더로 기분 입력 → 즉시 다음 단계로 자연 전환
- **가이드 프롬프트**: "오늘 뭐가 제일 좋았어?" 같은 질문으로 빈 화면 공포 제거
- **Daily Challenge**: 매일 달라지는 소규모 과제 (칭찬 3개 쓰기, 감사 1줄 쓰기 등)
- 음성-텍스트 변환 노트 입력 지원

### 진척/데이터 시각화

- **Mood Graph**: 주/월/연도별 감정 곡선
- **AI Weekly Insight**: 이번 주 감정 패턴 텍스트 요약 (AI 생성)
- **Habit Impact**: 특정 활동이 감정에 미친 영향 (Daylio와 유사하지만 AI 해석 포함)
- **Curated Feed**: 영감 인용구 피드 (수치가 아닌 감성적 콘텐츠 보조)

### 베일러 시스템모드 적용 포인트

- ✅ **슬라이더 입력**: 체크인 마찰을 극도로 낮추는 검증된 패턴
- ✅ **카드 분리 대시보드**: 프롬프트/챌린지/체크인을 분리된 카드로 → 인지 부하 낮춤
- ✅ **AI 인사이트 텍스트**: 그래프보다 한 줄 요약이 더 강한 리텐션 드라이버
- ✅ **Daily Challenge**: 관계 개선 행동 과제로 응용 가능

## 5. WOEBOT

> 출처: [woebothealth.com — Overview March 2024](https://woebothealth.com/img/2024/03/Woebot-Overview_March-2024.pdf) · [skywork.ai — Rise, Pivot, Legacy](https://skywork.ai/skypage/en/Woebot-Health:-The-Rise,-Pivot,-and-Legacy-of-an-AI-Therapy-Pioneer/1976115851798048768) · [medium.com/ai-in-mental-health — R.I.P. Woebot](https://medium.com/ai-in-mental-health/r-i-p-woebot-e0702ac9f771)

### ⚠️ 서비스 종료 확인

**2025년 6월 30일 B2C 서비스 완전 종료.** 이후 B2B(기업·의료기관) 전용으로 전환. 현재 앱 접근은 파트너 조직의 액세스 코드 필요.

### 홈 화면 구조 (종료 전)

- **채팅 창이 전부**: 전통적 메시지 앱 레이아웃
- 탭: Chat | Mood | Resources | Journal
- 첫 화면: Woebot의 "오늘 어떤 기분이야?" 인사 → 자연스러운 대화 시작
- 복잡한 네비게이션 없음 — 대화가 모든 기능의 진입점

### 핵심 인터랙션 패턴

- **선택지 기반 대화**: 자유 텍스트 + 다중선택지 병행 → AI 이해도 높이면서 마찰 낮춤
- **CBT 분기 플로우**:
  - 기분 수집 → 긍정: 심화 학습 콘텐츠 제안
  - 기분 수집 → 부정: 즉각 대처 도구 제공 (호흡, 인지 재구성 등)
- 모든 스크립트는 심리학자가 작성 (LLM 아님) → 예측 가능한 신뢰감
- **"열린 손" 철학**: 제안하되 강요하지 않음. 항상 "지금은 괜찮아" 옵션 존재

### 종료 교훈 (Medium 분석 기사 확인)

| 요인 | 내용 |
|------|------|
| B2C 확장의 한계 | 임상 수준 신뢰도를 B2C 마케팅으로 전달하기 어려움 |
| 규제 마찰 | FDA 인증 취득 과정의 비용과 속도 문제 |
| 리텐션 문제 | 위기 순간에만 사용하는 패턴 — 일상적 참여 낮음 |
| 시장 구조 | 임상 엄격함보다 편의성을 보상하는 B2C 시장 |
> "The direct-to-consumer dream is dimming, replaced by system integration and enterprise alignment" — Scott Wallace PhD

### 베일러 시스템모드 적용 포인트

- ✅ **감정 상태별 분기**: 점수 높을 때 vs 낮을 때 다른 화면/콘텐츠 제공
- ✅ **"열린 손" 원칙**: 항상 거절 옵션 제공 — 베일러 철학과 일치
- ⚠️ **종료 교훈**: 대화 전용 구조만으로는 일상적 참여 어려움 → 시스템모드는 대시보드 + 콘텐츠 조합 필요
- ⚠️ B2C 멘탈헬스 앱 단독으로는 지속 불가 → 베일러의 B2B 확장 전략과 일치

## 6. CALM

> 출처: [goodux.appcues.com — New User Experience](https://goodux.appcues.com/blog/calm-app-new-user-experience) · [usabilitygeek.com — UX Case Study](https://usabilitygeek.com/ux-case-study-calm-mobile-app/) · [support.calm.com — Weekly Progress Goal](https://support.calm.com/hc/en-us/articles/22317868359451) · [aws.amazon.com — Amazon Personalize case study](https://aws.amazon.com/blogs/machine-learning/personalizing-wellness-recommendations-at-calm-with-amazon-personalize/)

### 홈 화면 구조

- 배경: 산·물 풍경 이미지 (심리적 안정감 즉시 설정)
- 상단: 프로필 아이콘 + 설정 아이콘
- **하단 탭 3개**: Music | **Meditate (중앙·최대 크기)** | Sleep
- 홈 중앙: **Weekly Progress Goal 바** + 오늘 추천 콘텐츠
- 첫 방문: 4개의 tooltip overlay로 주요 섹션 안내 → 설명 없이도 시작 가능

### 핵심 인터랙션 패턴

**온보딩 플로우** (goodux 분석에서 확인):
1. "Take a deep breath" → 앱 목적 즉각 전달
2. 목표 선택 (Self-Esteem, Happiness, Stress 등) → 선언적 데이터 수집
3. 회원가입
4. **7일 무료 체험 제안** (결제 마찰 완화)
5. 알림 권한 요청
6. 개인화된 홈 구성 완료

- **1-tap 재생**: 콘텐츠 카드 탭 → 즉시 재생. 확인창 없음
- **tooltip 유도**: 처음 들어온 사용자에게 섹션별 간단한 안내 오버레이 (4개)
- **배경 자연음**: 콘텐츠 탐색 중에도 자연 소리 유지 → 분위기 연속성

### 진척/데이터 시각화

Calm Help Center 공식 문서에서 확인:
- **Weekly Progress Goal**: 홈 화면에 직접 표시되는 주간 목표 진행 바 (매주 리셋)
- **Dashboard** (프로필 → 우측 상단): 스트릭 + 총 세션 수 + 누적 명상 시간
- **월별 캘린더**: 명상한 날 표시
- 세션 완료 후 팝업: 현재 스트릭 + 누적 세션 수 + 총 명상 시간

**ML 기반 추천** (AWS Amazon Personalize 사례 연구):
- 사용자 행동 데이터로 다음 명상 자동 추천
- 스크롤 피로도 제거 + 콘텐츠 발견 시간 단축

### 베일러 시스템모드 적용 포인트

- ✅ **Weekly Progress Goal**: 홈 화면에 주간 목표 진행 바 직접 노출 → 베일러 "관계 건강도" 주간 목표에 응용
- ✅ **온보딩 퀴즈**: 시스템모드 첫 진입 시 "지금 집중하고 싶은 것" 선택 → 대시보드 개인화
- ✅ **1-tap 콘텐츠 진입**: Get 탭 콘텐츠를 즉시 시작 가능하게 설계
- ⚠️ 배경 이미지·자연음은 Calm의 감성 브랜드에 맞는 전략. 베일러 시스템모드는 쿨톤+미니멀로 차별화

## 9. VEILOR 시스템모드 UX 구조 제안

*이하는 조사 결과를 바탕으로 도출한 설계 제안입니다.*

### 9-1. 홈 화면 구조

```
┌──────────────────────────────────────┐
│  [Header]                            │
│  이름 · 오늘 ○월 ○일                 │
│  ───────────────────────────────── │
│  관계 건강도  ●●●●○  78점           │  ← Calm Weekly Progress Goal 응용
│  이번 주 목표 달성까지 +3점          │
├──────────────────────────────────────┤
│  [오늘 체크인]                       │
│  ○──────────────●───────○           │  ← Reflectly 슬라이더
│  오늘 전반적인 상태  (1~10)          │
│                                      │
│  [관계] [운동] [명상] [일] [혼자]   │  ← Daylio 활동 탭
│                           [저장]     │
├──────────────────────────────────────┤
│  [이번 주 스냅샷]                    │
│  감정 곡선 (7일 소형 그래프)         │  ← Reflectly Mood Graph
│  활동-감정 패턴 인사이트 1줄         │  ← Daylio Influence on Mood
├──────────────────────────────────────┤
│  [오늘 추천]                         │
│  ┌─────────────────────────────┐    │
│  │ 관계 챌린지: 오늘 5분 대화  │    │  ← Reflectly Daily Challenge
│  └─────────────────────────────┘    │
├──────────────────────────────────────┤
│  [AI 인사이트] (주 1회 갱신)         │
│  "화요일 저녁 감정 점수 패턴 발견"  │  ← Reflectly AI Insight
└──────────────────────────────────────┘
```

### 9-2. 플로우모드 대비 탭별 차이

| 탭 | 플로우모드 | 시스템모드 |
|----|-----------|-----------|
| **홈** | 엠버 인사 + 자유대화 | 대시보드 (지표+체크인+추천) |
| **Dig** | "자주 느끼는 것" 서술 카드 | 감정 트렌드 그래프 + 활동 상관관계 표 |
| **Get** | 엠버가 1가지 큐레이션 | 상태 기반 코스 카드 (Headspace 구조) |
| **Set** | 엠버가 Vent 기반 제안 | 목표 진행도 % + 다음 행동 제안 |
| **Me** | "이번 달의 나" 서술 | 수치 대시보드 + 월간 캘린더 히트맵 |

### 9-3. 리텐션 메커니즘 (검증된 것만)

| 메커니즘 | 근거 앱 | 베일러 적용 | 주의사항 |
|---------|---------|------------|---------|
| **누적 기록 강조** | Daylio | "총 12일 기록" > "12일 연속" | 연속 끊김 죄책감 절대 금지 |
| **AI 인사이트 텍스트** | Reflectly | 주 1회 패턴 분석 1줄 | 과잉 분석 피로 주의 |
| **Daily Challenge** | Reflectly | 관계 개선 소규모 과제 | 강제성 없음, 선택형 |
| **주간 목표 바** | Calm | 홈 화면 상단 고정 진행 바 | 매주 리셋 → 우상향 느낌 |
| **선형 코스 진행** | Headspace | 관계 개선 4주 코스 | 코스 이탈 방지 장치 필요 |
| **Adaptive 홈** | BetterUp | 감정 점수에 따라 다른 화면 | 기술 복잡도 고려 |

### 9-4. 설계 금기사항 (조사에서 도출)

1. **스트릭 끊김 = 실패** 느낌 → Headspace도 비판받는 패턴. 멘탈헬스 앱에서 수치심 유발
2. **과도한 데이터**: 지표 5개 이상 동시 노출 → 인지 부하. 핵심 지표 1개 + 보조 2개 이하
3. **빈 화면 공포**: 기록 없을 때 "아직 기록이 없습니다" 노출 금지 → 체크인 CTA 즉시 노출
4. **강제 알림**: 일 1회 초과 금지. 사용자 활동 시간대 1개만. 무시해도 앱이 벌주지 않음
5. **Woebot의 교훈**: 대화 전용 구조 + B2C만으로는 지속 불가. 콘텐츠+대시보드+코칭 조합 필요
