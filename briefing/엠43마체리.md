## M43 완성의 정의

다음이 모두 충족됐을 때 M43는 완성된다.

1. **이론 체계 완성** — 7개 프레임워크 + 231개 도메인 이론이 confirmed 상태
2. **연구원 콘텐츠 완성** — 각 도메인 최소 1개 Division Head 분석 존재
3. **세션 완성** — 각 핵심 도메인 밀라르쉬 × 연구원 토론 기록
4. **Veilrum 연결 완성** — m43_applications에 적용 추적 등록
5. **앱 연동 완성** — Veilrum 앱 실제 기능에 반영

---

## 작업 원칙 (순서 의존 관계)

```
LAYER 0 (인프라) → LAYER 1 (프레임워크) → LAYER 2 (핵심 5개) → LAYER 3 (Division 대표)
→ LAYER 4 (231개 전체) → LAYER 5 (검증) → LAYER 6 (앱 연동)
```

- **앞 LAYER 미완료 시 뒷 LAYER 작업 금지**
- **도메인 1개 완성 세트** = THEORY → ANALYSIS → SESSION → Veilrum 연결 → 검증 (이 순서 필수)
- **SESSION은 반드시 THEORY 완성 후**
- **이론 수정 시 m43_theory_revisions 기록 필수**

## 실행 도구 구분

| executor | 항목 수 | 해당 작업 |
|----------|---------|----------|
| `claude_code` | 70개 | LAYER 0~4 전체 + L5 교차 세션 — DB INSERT·파일 저장 필요 |
| `both` | 10개 | L5 검증(판단은 대화, status 업데이트는 코드) + L6 앱 연동(설계는 대화, 구현은 코드) |
