# M43 연구 콘텐츠 작업기획서
> Claude와 함께 M43 연구 콘텐츠를 생성할 때 참조하는 실행 가이드.
> **최종 업데이트:** 2026-03-10

---

## 1. 작업의 목적

M43 Institute가 연구하는 231개 도메인에 대해 밀라르쉬와 연구원들의 **실제 이론·분석·세션 콘텐츠**를 생성하고 DB에 저장한다.

이 콘텐츠는:
- M43 세계관을 실체 있는 연구 기관으로 만드는 자산
- Veilrum 플랫폼의 알고리즘·상담·콘텐츠의 원천 데이터
- 나중에 학술 출판물로 발전시킬 초안

---

## 2. 콘텐츠 생성 3가지 흐름

### 흐름 1 — 밀라르쉬 발제 이론 (THEORY)
- **DB 테이블:** `m43_domain_theories`
- **언어 스타일:** 철학적, 선구적, 질문을 던지는 방식
- **type:** `core_theory` | `framework` | `proposition` | `synthesis`
- **author_name:** '밀라르쉬' (researchers 테이블에 없음 — 의도적)

### 흐름 2 — 연구원 분석 (ANALYSIS)
- **DB 테이블:** `m43_research_outputs`
- **언어 스타일:** 학술적, 실증적, 선행 연구 참조
- **type:** `analysis` | `case_study` | `hypothesis` | `field_note` | `rebuttal`
- **연구원:** 해당 Division Head 또는 Specialist (캐릭터 배경 반영 필수)

### 흐름 3 — 세션 토론 (SESSION)
- **DB 테이블:** `m43_sessions` + `m43_session_contents`
- **형식:** 밀라르쉬 발제 → 연구원 반응·보완·반론 대화체
- **참여자:** 밀라르쉬 + 해당 Group Head 또는 Division Head

## 3. 도메인 1개 완성 세트

```
[도메인] 완성 세트 (순서)
STEP 1. THEORY   — 밀라르쉬 발제 (먼저)
STEP 2. ANALYSIS — Division Head 분석 (THEORY 기반)
STEP 3. SESSION  — 세션 토론 (THEORY + ANALYSIS 기반)
```

STEP 1만 먼저 만들고 다음 도메인으로 넘어가도 된다.
세트 완성은 나중에 해도 무방.

## 5. DB INSERT 표준 패턴

### THEORY INSERT
```sql
-- STEP 1: 도메인 ID 확인
SELECT id FROM m43_domains WHERE code = 'D1-001';

-- STEP 2: INSERT
INSERT INTO m43_domain_theories
  (domain_id, title, content, summary, type, status, tags)
VALUES (
  '{domain_id}',
  '제목',
  '본문 전체 (마크다운 제외, 텍스트)',
  '1~2문장 요약',
  'core_theory',
  'draft',
  ARRAY['태그1', '태그2']
)
RETURNING id;
```

### ANALYSIS INSERT
```sql
-- STEP 1: researcher_id 확인
SELECT id FROM m43_researchers WHERE name = 'Vera Lindgren';

-- STEP 2: INSERT
INSERT INTO m43_research_outputs
  (researcher_id, domain_id, title, content, summary, type, status, parent_theory_id, tags)
VALUES (
  '{researcher_id}',
  '{domain_id}',
  '제목',
  '본문',
  '요약',
  'analysis',
  'draft',
  '{theory_id}',
  ARRAY['태그']
)
RETURNING id;
```

### SESSION INSERT
```sql
-- sessions 먼저
INSERT INTO m43_sessions
  (title, type, topic, domain_ids, participant_ids, theory_ids, status)
VALUES (
  '세션명',
  'millarche_gh',  -- millarche_gh / millarche_dh / millarche_full
  '주제',
  ARRAY['{domain_id}'],
  ARRAY['{gh_id}'],
  ARRAY['{theory_id}'],
  'completed'
)
RETURNING id;

-- session_contents (발언 순서대로)
INSERT INTO m43_session_contents
  (session_id, speaker_name, sequence, content, type, referenced_domain_id, referenced_theory_id)
VALUES
  ('{session_id}', '밀라르쉬', 1, '발언', 'statement', '{domain_id}', '{theory_id}'),
  ('{session_id}', 'Nadia Kosovich', 2, '발언', 'response', '{domain_id}', NULL);
```

## 7. 콘텐츠 품질 기준

### 밀라르쉬 발제 (THEORY)
- 개인적 통찰에서 시작하는 도입부
- 기존 이론과의 차별점 명시
- M43이 검증해야 할 질문으로 마무리
- 분량: 1,500~3,000자

### 연구원 분석 (ANALYSIS)
- 해당 연구원의 전공·국적·성격 반영
- 선행 연구 2~3개 이상 언급 (실제 존재하는 연구)
- 밀라르쉬 이론을 학술 언어로 재구성
- 분량: 1,000~2,000자

### 세션 토론 (SESSION)
- 발언 5~10개 교환
- 반론 또는 보완이 반드시 포함
- 새로운 인사이트가 도출되는 구조
- 발언 당 100~300자
